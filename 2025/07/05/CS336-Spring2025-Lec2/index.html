背景

CS336 课程第二讲课堂笔记

问题

  1. 为什么要用 einops ？有什么好处？怎么用？

    • 相比于 index，可读性更好，review 时更容易理解
    • 三种操作：
      • einsum
      • rearrange
      • reduce

  2. 如何计算张量操作的计算量（用 flops 衡量）？

    • 用 flops 衡量（flop 是一次浮点计算，比如加，乘，注意 flops 和 flop/s 的区别）
    • MxN 和 NxK 矩阵乘需要多少 flops ?
      • 2 * M * N * K
      • 矩阵乘法存在优化，但是量级与上面计算一致
    • 因此前向传播的 FLOPS 是 2(#tokens)(#params)

  3. 什么是 Model FLOPs utilization？(MFU)

    • 模型 FLOPs 利用率 = 模型 FLOPs / 理论最大 FLOPs
    • 理论最大 FLOPs 是指模型在全精度下的 FLOPs

  4. 反向传播需要多少 FLOPS？

    • 反向传播的 FLOPS 是前向传播的 FLOPS 的两倍

  5. 参数初始化时采用 torch.randn 出现的问题？如何解决？

    • 与维度大小相关
      • 采用 Xavier 初始化

  6. 如何避免因为随机性导致的复现困难？

    • 固定随机种子:

  7. 整体数据太大无法全部加载到显存中怎么办？

    • lazily load, dataloader，一次加载一批数据

  8. 常见的优化器有哪些？

    • momentum = SGD + exponential averaging grad
    • AdaGrad = SGD + 梯度的平均
    • RMSProp = SGD + 梯度的指数平均
    • Adam = RMSProp + 动量

  9. AdaGrad 实现示范：

  10. 如何计算整个模型在训练时需要的 memory 和计算量（per step）?

    • 模型参数 + 激活参数 + 优化器参数 + 梯度参数
    • 见课件

  11. 如何使用混合精度？

    • 混合精度存在的意义在于一些计算需要高精度，而一些计算不需要，这时可以节省内存
    • 什么时候使用低精度，高精度？方法论？
      • 默认高精度，然后尽可能的向低精度靠拢
        • 低精度的问题在于训练不稳定（一些 trick 会有帮助）
      • pytorch 提供了混合精度的支持 AMP

总结

  1. 矩阵乘法占据了计算的大头，

    • 前向传播计算量 2(#tokens)(#params) = 2 m n p FLOPs
    • 反向传播计算量 4(#tokens)(#params) = 4 m n p FLOPs
    • 总计算量 2(#tokens)(#params) + 4(#tokens)(#params) = 6 m n p FLOPs

  2. FLOP/s 取决于硬件和数据类型(bf16)
