十、补充核心知识点

  1. 0/1背包问题核心：
    • 定义：有 n 个物品，每个物品只能选一次，背包容量为 C，求能装的最大价值（或是否能装满）
    • 状态转移（一维）：dp[j] = max(dp[j], dp[j - w[i]] + v[i])（求最大价值）或 dp[j] = dp[j] or dp[j - w[i]]（求是否能装满）
    • 遍历顺序：先物品后背包，倒序遍历背包，确保每个物品只选一次
  2. 子集和问题：
    • 是0/1背包的经典变体，目标是“是否能选出若干数，和恰好为 target”，直接套用0/1背包模板即可
  3. 完全背包与0/1背包的核心区别：
    • 完全背包：每个物品可以选无限次，遍历顺序是先物品后背包，正序遍历背包
    • 0/1背包：每个物品只能选一次，遍历顺序是先物品后背包，倒序遍历背包
  4. 动态规划优化的核心思路：
    • 若二维DP中 dp[i][j] 只依赖上一行 dp[i-1][...]，则可以压缩成一维DP，通过倒序遍历避免覆盖需要的历史状态，这是DP空间优化的常用手段

题目三：力扣32. 最长有效括号

难度：困难 | 标签：动态规划、栈、字符串 | 原题链接：https://leetcode.cn/problems/longest-valid-parentheses/

一、原题描述

给你一个只包含 '(' 和 ')' 的字符串，找出最长有效（格式正确且连续）括号子串的长度。

示例

示例 1： 输入：s = "(()" 输出：2 解释：最长有效括号子串是 "()"

示例 2： 输入：s = ")()())" 输出：4 解释：最长有效括号子串是 "()()"

示例 3： 输入：s = "" 输出：0

提示

  • 0 <= s.length <= 3 * 10^4
  • s[i] 为 '(' 或 ')'

二、我的初始思路&认识

我采用0-indexed的动态规划，定义 dp[i] 为以第 i 个字符结尾的最长有效括号子串的长度：

  • 如果 s[i] == '('，无法形成有效结尾，dp[i] = 0；
  • 如果 s[i] == ')'，分两种情况：
    1. 前一个字符是 '('（s[i-1] == '('），则 dp[i] = 2 + dp[i-2]（如果 i-2 >= 0）；
    2. 前一个字符是 ')'（s[i-1] == ')'），则找前一个有效子串前面的字符 s[i-dp[i-1]-1]，如果是 '('，则 dp[i] = dp[i-1] + 2 + dp[i-dp[i-1]-2]。 但我写代码时把匹配字符写反了，还有冗余条件，导致结果出错。

  2. 回文问题的核心性质 回文的核心是对称性，判断一个子串是否为回文，必须同时满足两个条件：① 子串两端的字符相等；② 去掉两端字符后的中间子串，也是回文。这也是回文问题天然适合用区间DP、中心扩展法解决的根本原因。

  4. 区间DP的通用遍历规则 区间DP的核心是「先解决短区间的子问题，再用短区间的结果解决长区间的问题」，因此遍历顺序必须按子串长度从小到大，而非按左边界、右边界的顺序遍历，否则会出现计算长区间时，内部子区间还未计算的问题。

  5. Manacher算法的核心优化点 它的线性复杂度来自于「利用回文的对称性，避免重复的扩展计算」。通过维护当前最右回文的边界，让大部分位置可以直接复用镜像点的回文半径，无需重新扩展，最终把均摊时间复杂度降到了O(n)。

  2. LCS状态转移的核心逻辑 LCS的转移逻辑可以用一句话总结：两个位置往前看，最后一个字符要么能配上，要么配不上。

    • 能配上：这个字符一定属于公共子序列，直接继承前 i-1 和前 j-1 的结果加1；
    • 配不上：这个字符不可能同时属于公共子序列，只能扔掉其中一个，取两种情况的最大值。

  3. 一维DP优化的通用条件 当二维DP中 dp[i][j] 仅依赖上一行的当前列（dp[i-1][j]）、当前行的前一列（dp[i][j-1]）、上一行的前一列（dp[i-1][j-1]）这三个值时，可以用一维DP优化空间。

    • 用 dp[j] 保存上一行的值；
    • 用 dp[j-1] 保存当前行前一列的值；
    • 用额外变量 prev 保存上一行前一列的值，避免被覆盖。

  4. DP初始化的物理意义 DP的初始化不是“随便设为0”，而是有明确的物理意义。本题中 dp[0][j] = 0 和 dp[i][0] = 0，物理意义是“当其中一个字符串为空时，最长公共子序列的长度一定是0”。初始化的正确性直接决定了后续转移的正确性，写DP前一定要先想清楚初始化的物理意义。

  5. LCS与其他双序列问题的关系 LCS是双序列DP的最基础模板题，很多其他双序列问题都是它的变体：

    • 编辑距离：在LCS的基础上，增加了“替换”操作，转移逻辑更复杂但核心思路一致；
    • 不相交的线：本质就是LCS，只是换了个题面描述；
    • 两个字符串的删除操作：可以通过LCS间接求解，删除次数 = 总长度 - 2*LCS长度。

    • dp[i][0]：word2为空，把word1前i个字符全删掉，所以操作数是i；

十、补充核心知识点

  1. 荷兰国旗问题的本质 不是“排序”，而是**“多指针原地分类”**——当元素种类极少时（比如本题3种，扩展题k种），不需要通用排序算法的“比较和交换”逻辑，只需要用k-1个指针维护k个区域的边界，把每个元素直接放到对应的区域即可，时间复杂度可以做到严格O(n)，且仅需一趟扫描。

  2. 三指针的边界维护细节

    • 遇到0时，交换的是 nums[i] 和 nums[l]，此时 nums[l] 要么是1（在1区），要么是0（已经在0区），所以交换后 i 可以直接右移，因为换过来的数已经处理过了；
    • 遇到2时，交换的是 nums[i] 和 nums[r]，此时 nums[r] 是未处理过的数，所以交换后 i 不能动，必须重新检查当前位置的新数；
    • 循环终止条件是 i <= r，因为当 i == r 时，还有一个未处理的元素，必须检查完。

  3. 为什么两次遍历法也是O(1)空间，但不是进阶最优 两次遍历法虽然空间复杂度也是O(1)，但需要两趟扫描，而荷兰国旗法只需要一趟扫描，在时间上更优（虽然都是O(n)，但常数因子更小），且更符合进阶要求的“一趟扫描”。

  4. 通用排序与分类问题的区别

    • 通用排序：适用于任意元素，需要通过比较确定元素的相对顺序，时间复杂度下限是O(nlogn)；
    • 分类问题：当元素种类极少时（比如本题只有3种），不需要比较相对顺序，只需要把元素放到对应的区域，时间复杂度可以做到O(n)。

题目十一：石头平衡分组问题（经典面试题/LeetCode 1049 简化版）

难度：中等 | 标签：数学、贪心、抵消思想 | 关联原题：1049. 最后一块石头的重量 II

一、原题描述

给定一个包含正整数的数组 nums，每个元素代表一块石头的重量。请你将这些石头分成两堆，使得两堆的总重量尽量平衡（即两堆重量差最小）。返回在最优情况下，每堆能达到的最大相同重量。

示例

示例 1： 输入：nums = [10,1,1,1] 输出：3 解释：最大石头10 > 其他所有石头之和3，因此最多只能让另一堆达到3，两堆分别为10和3。

示例 2： 输入：nums = [4,3,2,1] 输出：5 解释：总和为10，可以完全平衡分成两堆各5（如4+1和3+2）。

示例 3： 输入：nums = [2,7,4,1,8,1] 输出：11 解释：总和为23，最大石头8 ≤ 其他总和15，因此最多能达到23//2=11。

四、核心存在的问题

  1. 思路层面：对题目的强约束性质挖掘严重不足，只把它当成普通的子集和问题，没有发现“结果仅由总和和最大值两个量决定”这一关键数学观察。
  2. 认知层面：对“分两堆平衡”的本质理解不深——平衡的理论上限是总和的一半，但这个上限能否达到，取决于最大元素是否“过大”。
  3. 思维层面：存在“DP/背包万能”的思维定式，没有养成“先看有没有数学性质/贪心策略，再考虑通用算法”的解题习惯。
  4. 知识层面：对“石头碰撞/抵消/平衡分组”这一经典问题类型不熟悉，这类问题通常可以通过“最大值与其他总和的关系”快速简化。

五、惯性思路→最优解的思维转换方法

  1. 破局点：彻底放弃“先想DP”的惯性，先抓住“分两堆平衡”的本质——理论上的最大平衡重量是 sum//2，但这个上限能否达到，要看最大石头是否“过大”。
  2. 核心拐点：分两种情况讨论：
    • 情况1：最大石头过大——如果 max > sum - max（最大石头 > 其他所有石头之和），那么无论怎么分，最大石头必须单独在一堆，另一堆最多只能用其他所有石头，因此最大平衡重量是 sum - max；
    • 情况2：最大石头不大——如果 max <= sum - max，那么可以通过合理分配，让两堆尽量接近，达到理论上限 sum//2。
  3. 逻辑落地：最终答案就是两种情况的最小值——min(sum//2, sum - max)，因为：
    • 当最大石头过大时，sum - max < sum//2，取前者；
    • 当最大石头不大时，sum//2 <= sum - max，取前者。
  4. 细节兜底：如果题目要求返回“两堆的最小重量差”，则是 sum - 2 * min(sum//2, sum - max)，但本题直接返回最大平衡重量即可。

十、补充核心知识点

  1. 抵消/配对思想的通用套路 当题目中出现“分两堆”“平衡”“抵消”“配对”等关键词时，先问自己一个问题：最大元素是否超过其他所有元素的总和？

    • 如果是：最大元素无法被完全抵消/平衡，结果由最大元素决定；
    • 如果不是：可以通过合理分配，达到理论上的最优解（如总和的一半）。 这是一个非常高频的数学观察，能把很多O(n*sum)的DP问题直接降维到O(n)。

  2. 数学观察在算法题中的重要性 很多算法题（尤其是面试题）的最优解，不是靠复杂的算法（如DP、图论），而是靠对题目性质的深刻数学观察。 养成解题前先“停30秒，看看有没有数学性质/贪心策略”的习惯，往往能事半功倍。

  3. DP与数学结论的选择

    • 如果题目有明显的数学性质/贪心策略，且能严格证明正确性，优先用数学/贪心解法（时间空间更优）；
    • 如果没有明显的数学性质，再考虑通用算法（如DP、回溯）；
    • 面试时，如果能先给出数学/贪心最优解，再用DP作为对比/验证，会是很大的加分项。

  4. 本题与LeetCode 1049的关系 本题是1049的简化版：

    • 本题问“最大平衡重量”，答案是 min(sum//2, sum-max)；
    • 1049问“两堆的最小重量差”，答案是 sum - 2 * min(sum//2, sum-max)； 两者本质完全一致，只是问法不同。
