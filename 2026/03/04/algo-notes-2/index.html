算法刷题方法论记录（二）：滑窗、矩阵与惯性转换

我刷题最大的真实痛点不是“看不懂最优解”，而是：

  • 我能靠惯性写出一个版本（通常能过部分样例）
  • 但写不出稳定可复用的最优模板
  • 甚至卡在“循环怎么写”“边界怎么处理”“O(1) 空间怎么做到”这些实现层细节

所以我把目标改成：记住“思维惯性 → 最优解”的映射。 下次先写出惯性代码，再按映射把它机械转换成最优。

下面这 9 题就是我跟 AI 对话里反复出现的“典型卡点”。

题 1：LeetCode 3 — 无重复字符的最长子串

题意：在字符串中找到不含重复字符的最长子串长度。

我当时卡点 / 疑惑

  1. s[r] in dict 真的 O(1) 吗？
  2. dict.get() 为什么会误判？

关键认知补丁

  • s[r] in dict 平均 O(1)，最坏 O(n)，刷题默认平均分析。
  • dict.get(ch) 返回 0 时是 False，会把“出现过且位置=0”误判成“没出现过”。

惯性 → 最优映射

  • “用 get() 当存在判断” → “用 in 或 is not None”
  • “l 直接赋值 last+1” → “l 必须 max(l, last+1) 防止回退”

题 2：LeetCode 438 — 找到字符串中所有字母异位词

题意：返回 s 中所有与 p 互为异位词的子串起始下标。

我当时卡点 / 疑惑

  1. 我用“窗口排序再比较”，以为是 O(n log m)，但到底哪里不对？
  2. 我知道要滑动窗口，但“判断覆盖/相等”怎么做到 O(1)？
  3. 我维护 diff_sum/nonzero 时为什么会错？

关键认知补丁

  • 排序窗口每次是 O(m log m)，总共 O(n*m log m)，不是你以为的 log m * n。
  • 异位词判定的本质不是排序，而是频次向量相等。
  • 字符集固定 26：频次数组对比/维护可视为 O(1)（常数维度状态）。
  • 维护 nonzero（diff 中非 0 维度数）时，必须检查“变更前是否为 0”与“变更后是否为 0”，否则只会减不会加。

你总结的“误区”如何转成可执行规则

  1. a=ord('a')、m/n=len(...)：可以抽，但抽的是“稳定常量”，别把语义变量缩成无意义的短名。
  2. nonzero 更新：跨 0 才更新（状态机边界）。
  3. in/out：必须分块写成两个“原子更新”，别混在一起。

惯性 → 最优映射（你后续刷题最该背的一条）

  • “重算/排序” → “固定维度状态 + 每步 in/out 增量”
  • “只判断变更后是否为 0” → “变更前后都判断（跨 0 才改 nonzero）”
  • “in/out 混写” → “两次原子 update（可验证）”

最简模板（26 维差分 + nonzero）

题 3：LeetCode 76 — 最小覆盖子串

题意：找 s 的最短子串，使其包含 t 的所有字符（含重复）。

我当时卡点 / 疑惑

  1. 我知道滑窗：r 扩到覆盖、l 收到最小，但“是否覆盖”怎么高效？
  2. 如果 s[l] 不在 t 中，need[y] > 0 为什么不会误判 missing？

关键认知补丁

  • need[c] = count_t[c] - count_window[c]
  • missing 直接表示“还缺多少个字符（按数量）”，覆盖判定变成 missing==0（O(1)）
  • 不在 t 的字符：count_t=0，所以 need 永不为正；因此 need[y] > 0 等价于“我确实缺了 t 需要的字符”。

惯性 → 最优映射

  • “每步遍历 need 字典看是否还有 >0” → “missing 一个变量 O(1) 判覆盖”
  • “收缩时不知道何时停” → “当移出字符导致 need[y] > 0 时停止（刚好缺）”

最简模板

题 4：LeetCode 56 — 合并区间

题意：合并所有重叠区间。

我当时卡点 / 疑惑

我写“相邻比较”，用 intervals[r-1][1] 判断断开，结果遇到“长区间覆盖短区间”就错了。

关键认知补丁

关键不是维护“最左最右”，而是维护 当前合并段 active interval 的 running end。 排序后，后面的区间只可能把右端点推得更远，不会影响左端点。

惯性 → 最优映射

  • “看相邻区间是否重叠” → “看 next.start 是否超过当前合并段 cur_end”
  • “合并段 end 用 r-1” → “合并段 end 永远用 max(cur_end, next_end)”

最简模板

题 5：LeetCode 238 — 除自身以外数组的乘积

题意：输出每个位置除自身以外的乘积，不能用除法。

我当时卡点 / 疑惑

我构造“包含自身”的前缀和后缀，导致边界分支一堆，还想去改原数组当后缀缓存。

关键认知补丁

真正的核心不是“少用数组”，而是把“前缀/后缀”抽象成 滚动状态变量。 用乘法单位元 1 吸收边界，就不需要 if i==0。

惯性 → 最优映射

  • “前/后缀数组（依赖下标）” → “prefix/suffix 滚动变量（不变量驱动）”
  • “if 处理边界” → “单位元 1 消边界”

最简模板

题 6：LeetCode 41 — 缺失的第一个正数

题意：找数组中缺失的最小正整数，要求 O(n) 时间、O(1) 额外空间。

我当时卡点 / 疑惑

O(1) 空间把 set/排序都禁了，完全没思路。

关键认知补丁

把数组当哈希桶：值 x 应该去下标 x-1。 交换把值送回正确位置，最后扫第一个不匹配的位置。

最容易错的点（你抓得很准）：交换后不能 i++，因为 i 位置来了新值需要继续处理。

题 7：LeetCode 73 — 矩阵置零

题意：若某元素为 0，则其所在行列全置 0，要求原地（O(1) 空间）。

我当时卡点 / 疑惑（你最核心的疑惑点）

  1. 我会记 rows/cols（O(m+n)），但 O(1) 怎么做？
  2. 用第一行第一列当标记会覆盖原数据——如果第一行第一列不该清零，怎么恢复？

最关键的解决方案（必须写清楚）

解决“覆盖恢复不了”的关键不是恢复，而是：提前保存边界信息。

  • 我们用 matrix[i][0] 作为 “第 i 行要清零” 的标记位
  • 用 matrix[0][j] 作为 “第 j 列要清零” 的标记位 这必然会覆盖第一行/第一列的某些格子。

所以必须在写标记之前保存：

  • first_row_zero：第一行原本是否含 0
  • first_col_zero：第一列原本是否含 0

为什么不需要恢复？

  • 第一行最终是否全 0，只取决于它“原本是否含 0”（first_row_zero）。
  • 第一列最终是否全 0，只取决于它“原本是否含 0”（first_col_zero）。
  • 标记阶段写入的 matrix[0][j]=0 只会发生在“第 j 列本来就该清零”的情况下，因此最终 (0,j) 本来就应为 0，不存在“误伤需要恢复”的需求。

惯性 → 最优映射（矩阵题高频模板）

  • “开 rows/cols 标记数组” → “复用 matrix 第一行/列当标记位”
  • “担心覆盖恢复不了” → “保存两条边界信息 first_row/first_col；不恢复，只在最后按布尔清边界”
  • “读写混在一起” → “固定四阶段：读边界 → 写标记 → 清内部 → 清边界”

最简模板（含 any 的转换）

题 8：LeetCode 54 — 螺旋矩阵

题意：顺时针螺旋输出矩阵所有元素。

我当时卡点 / 疑惑（你强调必须写清楚的）

  1. 我连循环怎么写都不知道：螺旋怎么“转”太抽象。
  2. 边界条件如何简洁：总是担心重复遍历单行/单列。

核心思路（把抽象变成“骨架”）

别想“螺旋怎么转”，只想“这一层矩形怎么走四条边”：

  • 当前未遍历区域永远是闭区间框：行 [top, bottom]，列 [left, right]

  • 每轮固定走：上边 → 右边 → 下边 → 左边，然后收缩边界

  • 为了避免只剩一行/一列时重复遍历，下边和左边前需要两个判定：

    • if top <= bottom 才走下边
    • if left <= right 才走左边

口诀：

闭区间框住未遍历区域；每轮上右下左走一圈并收缩；下/左两步用 top<=bottom、left<=right 防重复。
