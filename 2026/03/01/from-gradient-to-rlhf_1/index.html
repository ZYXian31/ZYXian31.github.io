从梯度到 RLHF：策略梯度的数学本质、方差来源与工程控制（Part I）

0. 预备说明：确定性梯度 vs 随机梯度估计器

目的：解决“梯度为什么会有方差？”这一误解

很多读者（包括我最初讨论时的直觉）会认为：梯度是反向传播算出来的，既然计算图确定、数值确定，梯度怎么会有方差？

这类疑问的关键在于：我们讨论的是两件不同的东西：

  • 确定性梯度：对一个确定的目标函数（以及确定的数据）求导，结果是确定的数。
  • 随机梯度估计器：目标函数是一个期望（Expectation），我们用随机采样来估计这个期望的梯度。此时“梯度估计值”本身是随机变量，因此有方差。

下面把这两者严格区分开。

0.1 固定数据下的梯度是确定的

以监督学习为例。给定一个固定的数据集 ($D={(xi,y_i)}{i=1}^N$)，定义经验风险：

  • 数据集 ($D$) 是固定的（常量）

注意：工程上我们常说 “SGD 的梯度有噪声/方差”，那是因为我们用 mini-batch 近似全量数据梯度；但对一个固定 mini-batch，梯度依然是确定的。
也就是说：监督学习里“随机性”来自我们人为抽样 mini-batch，而不是来自模型输出必须被采样。

  • 回报：($R(\tau)$) 依赖于随机轨迹，因此也是随机变量

0.2.2 为什么固定数据下梯度就不是随机变量，而策略梯度是？

对比的关键在于：“输入是否是随机生成的”。

  • 监督学习（固定数据）：你计算 ($\nabla_\theta L(\theta)$) 时，参与计算的样本 (${(x_i,y_i)}$) 是固定常量；梯度是确定函数值。
  • 策略梯度（采样轨迹）：你计算 ($g(\tau)$) 时，参与计算的是由当前策略与环境交互产生的随机轨迹 ($\tau$)。即使你固定 ($\theta$)，每次 rollout 仍会产生不同的 ($\tau$)，从而得到不同的 ($g(\tau)$)。

更直白地说：

  • 监督学习：样本在磁盘里，训练时只是读取
  • 强化学习 / RLHF：样本要“生成出来”（rollout / sampling），生成过程本身带随机性

所以两者的“随机性地位”不同：

  • 监督学习里，随机性通常是你为了省计算故意抽 mini-batch（可关掉）
  • 策略梯度里，随机性来自策略的随机采样与环境随机性（通常关不掉，除非你用完全确定性策略且环境确定，且仍然存在探索/可导性问题）

0.3 工程视角：SFT 与 RLHF 中梯度随机性的本质差别

理解策略梯度为什么有方差，最直观的方法就是对比：

  • SFT（Supervised Fine-Tuning）
  • RLHF（基于策略梯度的微调）

它们都训练同一个模型，但梯度的随机性来源完全不同。

0.3.6 一句话总结

SFT 的梯度是“对固定数据的确定性函数求导”；
RLHF 的梯度是“对随机生成样本的期望进行 Monte Carlo 估计”。

因此：

  • 策略梯度的方差不是数值误差
  • 不是实现问题
  • 而是优化目标本身的统计性质决定的

这也是后文所有“baseline、advantage、PPO、KL 控制”的理论前提。

本节小结

  • 对固定数据的梯度是确定的：随机性只来自你是否抽 mini-batch。
  • 策略梯度训练时用的是“随机梯度估计器” ($g(\tau)$)，随机性来自轨迹/文本的采样。
  • 在 LLM-RLHF 中，即使 reward model 是确定函数，reward 作为随机变量仍然会因生成样本不同而波动。
  • 期望梯度无法解析计算，原因是轨迹/序列空间指数爆炸 + 环境黑箱，只能用采样估计，因此方差是内生的。

1. 梯度与策略梯度：定义与区别

这一节的目标是把三个概念彻底分开：梯度（纯数学对象）、用梯度做优化（下降/上升只是符号选择）、以及策略梯度（强化学习里对“分布参数”求导的特定形式）。后面的推导、方差分析、baseline 和 PPO/KL 都建立在这三者的清晰区分之上。

1.3 策略梯度定义

强化学习中，我们的“模型”不是直接输出一个数，而是输出一个策略（policy）：给定状态 (s)，输出对动作 (a) 的概率分布。

其中 ($\theta$) 是策略参数（例如神经网络参数），($\pi_\theta(a|s)$) 是在状态 ($s$) 选择动作 ($a$) 的概率。

强化学习的优化目标通常写成期望回报：

  • ($\tau$) 是轨迹（trajectory），包含状态、动作序列：($(s_0,a_0,s_1,a_1,\dots)$)
  • ($P\theta(\tau)$) 是在策略 ($\pi\theta$) 下产生轨迹的概率分布
  • ($R(\tau)$) 是该轨迹的回报（return）

于是，“策略梯度（policy gradient）”指的就是：

也就是：对策略参数求导，让期望回报变大。

和监督学习最重要的不同点在于：
这里的目标函数是一个对“轨迹分布”取期望的量，($\theta$) 影响的是 ($P_\theta(\tau)$)（即采样分布），而不是直接给出一个可微的标量输出供你对齐标签。

在 LLM 的 RLHF 里，策略就是语言模型的 token 分布：

轨迹是完整生成序列 ($y=(y_1,\dots,y_T)$)，回报是 reward model 对整段输出的评分 ($r(y)$)。形式完全一致：

1.4 监督学习 vs 强化学习的差异

这一节只强调最核心、与后文推导直接相关的差异：优化对象、数据生成机制、以及梯度信号的来源。

1.4.1 优化对象不同：拟合标签 vs 优化期望回报

  • 监督学习：数据集提供“正确答案/标签”，目标是最小化预测误差
  • 强化学习：没有逐步标签，目标是最大化交互产生的期望回报

监督学习里的误差信号来自“标签差”，强化学习里来自“回报好坏”。

1.4.2 数据生成机制不同：离线固定数据 vs 在线采样轨迹

  • 监督学习：样本 ((x,y)) 在数据集里是固定的，你读取它们并计算梯度
  • 强化学习：样本（轨迹）需要由当前策略去环境中生成/采样出来

这一点决定了：强化学习里你讨论的不是“对固定样本的梯度”，而是“对随机采样轨迹的梯度估计器”（这也是为什么后文必须谈方差、baseline、PPO 等稳定化技术）。

1.4.3 可微性链路不同：计算图可见 vs 黑箱环境/采样节点

  • 监督学习的计算图通常是：

    全部在模型内部，端到端可微。

  • 强化学习的交互链路通常是：

    其中“采样动作”和“环境转移”通常不可微或不可得解析形式。

因此强化学习要用“对分布求导”的方式（后文的 log-likelihood trick / score function）把梯度写成可采样估计的形式。

1.4.4 信号结构不同：稠密监督 vs 延迟回报（信用分配）

监督学习通常每个样本都有直接误差信号；强化学习常见设置是回报在轨迹末端或稀疏出现，导致信用分配（credit assignment）更难、方差更大。LLM 的 RLHF 就属于典型的“序列末端给分”的结构：reward 对整段生成打分，但梯度要分摊到每个 token 的采样决策上。

本节小结

  • 梯度是局部一阶变化率向量；下降/上升仅由目标的最小化/最大化决定。
  • 策略梯度是强化学习中对 ($J(\theta)=\mathbb{E}[R]$) 求导，其核心是：($\theta$) 改变的是采样分布而非直接标签误差。
  • 监督学习与强化学习的关键差异在于数据生成与可微链路：前者“固定数据 + 直接误差”，后者“采样轨迹 + 分布求导”，这自然引出后文的 log-likelihood trick 与方差控制工具。
