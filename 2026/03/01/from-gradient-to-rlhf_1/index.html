从梯度到 RLHF：策略梯度的数学本质、方差来源与工程控制（Part I）

0. 预备说明：确定性梯度 vs 随机梯度估计器

目的：解决“梯度为什么会有方差？”这一误解

很多读者（包括我最初讨论时的直觉）会认为：梯度是反向传播算出来的，既然计算图确定、数值确定，梯度怎么会有方差？

这类疑问的关键在于：我们讨论的是两件不同的东西：

  • 确定性梯度：对一个确定的目标函数（以及确定的数据）求导，结果是确定的数。
  • 随机梯度估计器：目标函数是一个期望（Expectation），我们用随机采样来估计这个期望的梯度。此时“梯度估计值”本身是随机变量，因此有方差。

下面把这两者严格区分开。

0.1 固定数据下的梯度是确定的

以监督学习为例。给定一个固定的数据集 (D={(xi,yi)}i=1ND=\{(x_i,y_i)\}_{i=1}^ND={(xi​,yi​)}i=1N​)，定义经验风险：

L(θ)=1N∑i=1Nℓ(fθ(xi),yi)L(\theta)=\frac{1}{N}\sum_{i=1}^{N}\ell\big(f_\theta(x_i),y_i\big) L(θ)=N1​i=1∑N​ℓ(fθ​(xi​),yi​)

这里：

  • 数据集 (DDD) 是固定的（常量）
  • 模型 (fθf_\thetafθ​) 是确定的可微函数
  • 损失 (ℓ(⋅)\ell(\cdot)ℓ(⋅)) 是确定的可微函数

因此，梯度

∇θL(θ)\nabla_\theta L(\theta) ∇θ​L(θ)

在给定 (DDD) 与 (θ\thetaθ) 时是一个确定的向量：你在同一台机器上重复计算（忽略浮点误差）会得到同一个值。

注意：工程上我们常说 “SGD 的梯度有噪声/方差”，那是因为我们用 mini-batch 近似全量数据梯度；但对一个固定 mini-batch，梯度依然是确定的。 也就是说：监督学习里“随机性”来自我们人为抽样 mini-batch，而不是来自模型输出必须被采样。

0.2 策略梯度是随机变量

强化学习与 LLM-RLHF 里，目标通常是最大化期望回报：

J(θ)=Eτ∼Pθ(τ)[R(τ)]J(\theta)=\mathbb{E}_{\tau\sim P_\theta(\tau)}[R(\tau)] J(θ)=Eτ∼Pθ​(τ)​[R(τ)]

这里出现了本节核心对象：(τ\tauτ)（trajectory，轨迹）。

0.2.1 哪些东西是随机变量？

随机变量不是 (∇θJ(θ)\nabla_\theta J(\theta)∇θ​J(θ)) 这个数学对象本身，而是我们在实践中用来估计它的“单次采样梯度”：

  • 轨迹：(τ=(s0,a0,s1,a1,… )\tau=(s_0,a_0,s_1,a_1,\dots)τ=(s0​,a0​,s1​,a1​,…)) 是随机变量
  • 回报：(R(τ)R(\tau)R(τ)) 依赖于随机轨迹，因此也是随机变量
  • 单次采样的梯度估计器：

g(τ)=R(τ)∇θlog⁡Pθ(τ)g(\tau)=R(\tau)\nabla_\theta\log P_\theta(\tau) g(τ)=R(τ)∇θ​logPθ​(τ)

是随机变量（因为它是随机轨迹 (τ\tauτ) 的函数）。

真正想要的梯度是期望：

∇θJ(θ)=Eτ∼Pθ[g(τ)]\nabla_\theta J(\theta)=\mathbb{E}_{\tau\sim P_\theta}[g(\tau)] ∇θ​J(θ)=Eτ∼Pθ​​[g(τ)]

而训练时我们拿到的是有限条采样轨迹的平均：

∇θJ^=1N∑i=1Ng(τi),τi∼Pθ\widehat{\nabla_\theta J}=\frac{1}{N}\sum_{i=1}^N g(\tau_i) \quad,\quad \tau_i\sim P_\theta ∇θ​J​=N1​i=1∑N​g(τi​),τi​∼Pθ​

由于 (τi\tau_iτi​) 是随机采样的，这个估计 (∇θJ^\widehat{\nabla_\theta J}∇θ​J​) 自然就会在不同 batch 之间波动——这就是“方差”的来源。

0.2.2 为什么固定数据下梯度就不是随机变量，而策略梯度是？

对比的关键在于：“输入是否是随机生成的”。

  • 监督学习（固定数据）：你计算 (∇θL(θ)\nabla_\theta L(\theta)∇θ​L(θ)) 时，参与计算的样本 ({(xi,yi)}\{(x_i,y_i)\}{(xi​,yi​)}) 是固定常量；梯度是确定函数值。
  • 策略梯度（采样轨迹）：你计算 (g(τ)g(\tau)g(τ)) 时，参与计算的是由当前策略与环境交互产生的随机轨迹 (τ\tauτ)。即使你固定 (θ\thetaθ)，每次 rollout 仍会产生不同的 (τ\tauτ)，从而得到不同的 (g(τ)g(\tau)g(τ))。

更直白地说：

  • 监督学习：样本在磁盘里，训练时只是读取
  • 强化学习 / RLHF：样本要“生成出来”（rollout / sampling），生成过程本身带随机性

所以两者的“随机性地位”不同：

  • 监督学习里，随机性通常是你为了省计算故意抽 mini-batch（可关掉）
  • 策略梯度里，随机性来自策略的随机采样与环境随机性（通常关不掉，除非你用完全确定性策略且环境确定，且仍然存在探索/可导性问题）

0.2.3 随机性具体体现在哪里？从什么分布采样？

要回答“从什么分布采样”，我们把轨迹分布 (Pθ(τ)P_\theta(\tau)Pθ​(τ)) 展开：

Pθ(τ)=ρ(s0)∏t=0T−1πθ(at∣st) P(st+1∣st,at)P_\theta(\tau) = \rho(s_0)\prod_{t=0}^{T-1}\pi_\theta(a_t|s_t)\,P(s_{t+1}|s_t,a_t) Pθ​(τ)=ρ(s0​)t=0∏T−1​πθ​(at​∣st​)P(st+1​∣st​,at​)

其中：

  • (ρ(s0)\rho(s_0)ρ(s0​))：初始状态分布
  • (πθ(at∣st)\pi_\theta(a_t|s_t)πθ​(at​∣st​))：策略分布（由模型给出）
  • (P(st+1∣st,at)P(s_{t+1}|s_t,a_t)P(st+1​∣st​,at​))：环境转移分布（黑箱或仿真器）

这告诉我们：轨迹的随机性至少来自两处：

  1. 动作采样：at∼πθ(⋅∣st)a_t\sim \pi_\theta(\cdot|s_t)at​∼πθ​(⋅∣st​)
  2. 环境随机性：(st+1∼P(⋅∣st,at)s_{t+1}\sim P(\cdot|s_t,a_t)st+1​∼P(⋅∣st​,at​))

在 LLM-RLHF 中，环境可以理解为“生成过程 + reward 评价”，最核心的随机性来自：

yt∼πθ(⋅∣y<t,x)y_t \sim \pi_\theta(\cdot|y_{<t},x) yt​∼πθ​(⋅∣y<t​,x)

即每个 token 的采样。

即使 reward model 是确定函数 (r(y)r(y)r(y))，由于 (yyy) 是从 (πθ\pi_\thetaπθ​) 随机采样得到，(R=r(y)R=r(y)R=r(y)) 作为随机变量仍然会有方差：

Var(R)=Vary∼πθ[r(y)]\mathrm{Var}(R)=\mathrm{Var}_{y\sim\pi_\theta}[r(y)] Var(R)=Vary∼πθ​​[r(y)]

0.2.4 一个极简数值例子：单 token 的梯度为什么会波动？

很多人（包括我在讨论中）会误以为“梯度波动来自数值精度”。实际上，不需要任何数值误差，采样本身就足以制造方差。

假设某一步 token 只有两个候选 A / B：

π(A)=0.9,π(B)=0.1\pi(A)=0.9,\quad \pi(B)=0.1 π(A)=0.9,π(B)=0.1

把参数简化为控制 (π(B)\pi(B)π(B)) 的 logit (zBz_BzB​)。则（softmax/sigmoid 的标准结果）：

  • 如果采到 B：

u=∇zBlog⁡π(B)=1−π(B)=0.9u=\nabla_{z_B}\log\pi(B)=1-\pi(B)=0.9 u=∇zB​​logπ(B)=1−π(B)=0.9

  • 如果采到 A：

u=∇zBlog⁡π(A)=−π(B)=−0.1u=\nabla_{z_B}\log\pi(A)=-\pi(B)=-0.1 u=∇zB​​logπ(A)=−π(B)=−0.1

因此 (uuu) 是随机变量：

u={0.9概率 0.1−0.1概率 0.9u= \begin{cases} 0.9 & \text{概率 }0.1 \\ -0.1 & \text{概率 }0.9 \end{cases} u={0.9−0.1​概率 0.1概率 0.9​

它的期望是 0（这是一个重要恒等式）：

E[u]=0.1⋅0.9+0.9⋅(−0.1)=0\mathbb{E}[u]=0.1\cdot 0.9 + 0.9\cdot (-0.1)=0 E[u]=0.1⋅0.9+0.9⋅(−0.1)=0

但二阶矩非零：

E[u2]=0.1⋅0.92+0.9⋅0.12=0.1⋅0.81+0.9⋅0.01=0.09\mathbb{E}[u^2]=0.1\cdot 0.9^2+0.9\cdot 0.1^2 =0.1\cdot 0.81+0.9\cdot 0.01 =0.09 E[u2]=0.1⋅0.92+0.9⋅0.12=0.1⋅0.81+0.9⋅0.01=0.09

所以方差：

Var(u)=0.09\mathrm{Var}(u)=0.09 Var(u)=0.09

这说明：即便完全没有数值误差，单 token 的 log-prob 梯度也会因采样而波动。

一旦你把多个 token 的梯度相加、再乘上回报 (RRR)，方差会进一步累积和放大——这就是后文“策略梯度方差大”的根源之一。

0.2.5 为什么不能“解析地”计算期望梯度，而必须采样估计？

你可能会问：既然

∇θJ(θ)=Eτ∼Pθ[g(τ)]\nabla_\theta J(\theta)=\mathbb{E}_{\tau\sim P_\theta}[g(\tau)] ∇θ​J(θ)=Eτ∼Pθ​​[g(τ)]

那我能不能把这个期望直接算出来？

理论上，如果你能枚举所有轨迹 (τ\tauτ)，当然可以：

∇θJ(θ)=∑τPθ(τ)g(τ)\nabla_\theta J(\theta)=\sum_\tau P_\theta(\tau)g(\tau) ∇θ​J(θ)=τ∑​Pθ​(τ)g(τ)

但在实际问题里，这条路基本不可行，原因是轨迹空间爆炸：

  • 离散动作空间大小为 (∣A∣|\mathcal{A}|∣A∣)
  • 序列/轨迹长度为 (TTT)

仅动作序列数就有：

∣A∣T|\mathcal{A}|^T ∣A∣T

在 LLM 中更夸张：

  • token 词表 (∣V∣|V|∣V∣) 可能是 50k
  • 序列长度 (TTT) 动辄几百上千

序列数 (∣V∣T|V|^T∣V∣T) 天文级，无法枚举，也无法精确求和。

同时，轨迹分布还包含环境转移 (P(s′∣s,a)P(s'|s,a)P(s′∣s,a))，真实世界或复杂仿真里它是黑箱，你甚至没有解析形式可用。

因此实际只能用 Monte Carlo：

∇θJ(θ)≈1N∑i=1Ng(τi),τi∼Pθ\nabla_\theta J(\theta) \approx \frac{1}{N}\sum_{i=1}^{N} g(\tau_i) \quad,\quad \tau_i\sim P_\theta ∇θ​J(θ)≈N1​i=1∑N​g(τi​),τi​∼Pθ​

这一步把“解析求和”换成“随机采样平均”，代价就是：

  • 估计器有方差
  • 方差随 (NNN) 增大而下降（大致 (∝1/N\propto 1/N∝1/N)），但样本成本很高

0.3 工程视角：SFT 与 RLHF 中梯度随机性的本质差别

理解策略梯度为什么有方差，最直观的方法就是对比：

  • SFT（Supervised Fine-Tuning）
  • RLHF（基于策略梯度的微调）

它们都训练同一个模型，但梯度的随机性来源完全不同。

0.3.1 SFT：Teacher Forcing 下的“条件确定性梯度”

在 SFT 中，我们有固定的数据对：

(x,y1,y2,…,yT)(x, y_1, y_2, \dots, y_T) (x,y1​,y2​,…,yT​)

训练目标是最大化条件似然：

L(θ)=∑t=1Tlog⁡πθ(yt∣y<t,x)L(\theta) = \sum_{t=1}^{T} \log \pi_\theta(y_t \mid y_{<t}, x) L(θ)=t=1∑T​logπθ​(yt​∣y<t​,x)

梯度为：

∇θL(θ)=∑t=1T∇θlog⁡πθ(yt∣y<t,x)\nabla_\theta L(\theta) = \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(y_t \mid y_{<t}, x) ∇θ​L(θ)=t=1∑T​∇θ​logπθ​(yt​∣y<t​,x)

关键点：

  • 所有 token (y_t) 是数据集给定的固定值
  • 不存在采样
  • 给定 mini-batch 后梯度是确定的

即使模型输出分布是概率分布，我们并没有从中采样，而是用真实标签（teacher forcing）直接计算 log-prob。

因此：

  • SFT 的随机性仅来自“我们抽取了哪个 mini-batch”
  • 对于固定 mini-batch，梯度是确定的

0.3.2 RLHF：必须采样，随机性不可消除

在 RLHF 中，我们优化：

J(θ)=Ey∼πθ(⋅∣x)[r(y)]J(\theta) = \mathbb{E}_{y \sim \pi_\theta(\cdot|x)} [r(y)] J(θ)=Ey∼πθ​(⋅∣x)​[r(y)]

训练步骤是：

  1. 用当前模型 采样生成文本：

y∼πθ(⋅∣x)y \sim \pi_\theta(\cdot|x) y∼πθ​(⋅∣x)

  1. 用 reward model 计算分数：

R=r(y)R = r(y) R=r(y)

  1. 用策略梯度更新：

g=R∑t=1T∇θlog⁡πθ(yt∣y<t,x)g = R \sum_{t=1}^{T} \nabla_\theta \log \pi_\theta(y_t|y_{<t},x) g=Rt=1∑T​∇θ​logπθ​(yt​∣y<t​,x)

注意这里的根本差异：

  • 在 SFT 中，(y_t) 来自数据集（常量）
  • 在 RLHF 中，(y_t) 来自模型分布（随机变量）

因此：

  • 每一次 rollout 都可能生成不同文本
  • reward 会随生成文本变化
  • 每个 token 的 log-prob 梯度也随采样 token 改变

即使：

  • reward model 是确定函数
  • 环境是确定性的

只要生成是采样的：

y∼πθy \sim \pi_\theta y∼πθ​

梯度估计器就不可避免是随机变量。

0.3.3 一个极简对比示例

假设 prompt 相同：

SFT 情况

数据给定：

训练时永远计算：

∇θlog⁡πθ(4∣2+2=?)\nabla_\theta \log \pi_\theta(4|\text{2+2=?}) ∇θ​logπθ​(4∣2+2=?)

梯度是确定的。

RLHF 情况

模型可能生成：

  • “4”（reward = 1）
  • “5”（reward = 0）
  • “The answer is 4.”（reward = 0.9）

不同采样 → 不同 reward → 不同梯度。

哪怕参数 θ 不变，两次 rollout 得到的梯度也不同。

0.3.4 为什么 RLHF 不能改成“像 SFT 一样不采样”？

因为 RLHF 的目标不是拟合固定标签，而是最大化：

Ey∼πθ[r(y)]\mathbb{E}_{y\sim \pi_\theta}[r(y)] Ey∼πθ​​[r(y)]

如果不采样，就无法估计模型在当前分布下的期望 reward。

你无法提前知道：

  • 哪些输出当前策略概率高
  • 哪些输出 reward 高

因此必须从当前策略分布中采样，估计：

∇θJ(θ)\nabla_\theta J(\theta) ∇θ​J(θ)

这一步不可避免地引入 Monte Carlo 估计误差。

0.3.5 核心对比总结

SFT RLHF
token 来源 数据集 模型采样
reward 来源 固定标签 reward model
梯度是否条件确定 是（给定 batch） 否
方差来源 mini-batch 抽样 生成采样 + reward 波动

0.3.6 一句话总结

SFT 的梯度是“对固定数据的确定性函数求导”； RLHF 的梯度是“对随机生成样本的期望进行 Monte Carlo 估计”。

因此：

  • 策略梯度的方差不是数值误差
  • 不是实现问题
  • 而是优化目标本身的统计性质决定的

这也是后文所有“baseline、advantage、PPO、KL 控制”的理论前提。

本节小结

  • 对固定数据的梯度是确定的：随机性只来自你是否抽 mini-batch。
  • 策略梯度训练时用的是“随机梯度估计器” (g(τ)g(\tau)g(τ))，随机性来自轨迹/文本的采样。
  • 在 LLM-RLHF 中，即使 reward model 是确定函数，reward 作为随机变量仍然会因生成样本不同而波动。
  • 期望梯度无法解析计算，原因是轨迹/序列空间指数爆炸 + 环境黑箱，只能用采样估计，因此方差是内生的。

1. 梯度与策略梯度：定义与区别

这一节的目标是把三个概念彻底分开：梯度（纯数学对象）、用梯度做优化（下降/上升只是符号选择）、以及策略梯度（强化学习里对“分布参数”求导的特定形式）。后面的推导、方差分析、baseline 和 PPO/KL 都建立在这三者的清晰区分之上。

1.1 梯度定义

1.1.1 数学定义（严格形式）

设有一个从参数空间到实数的可微函数：

f:Rd→R,f=f(θ)f:\mathbb{R}^d \rightarrow \mathbb{R},\quad f=f(\theta) f:Rd→R,f=f(θ)

它在点 (θ\thetaθ) 的梯度定义为偏导数组成的向量：

∇θf(θ)=(∂f∂θ1,∂f∂θ2,…,∂f∂θd)⊤\nabla_\theta f(\theta) = \left( \frac{\partial f}{\partial \theta_1}, \frac{\partial f}{\partial \theta_2}, \dots, \frac{\partial f}{\partial \theta_d} \right)^\top ∇θ​f(θ)=(∂θ1​∂f​,∂θ2​∂f​,…,∂θd​∂f​)⊤

它满足一个关键的“局部线性近似”性质：对任意小扰动 (Δθ\Delta \thetaΔθ)，有

f(θ+Δθ)≈f(θ)+∇θf(θ)⊤Δθf(\theta+\Delta\theta) \approx f(\theta) + \nabla_\theta f(\theta)^\top \Delta\theta f(θ+Δθ)≈f(θ)+∇θ​f(θ)⊤Δθ

这句话的意义是：在足够小的邻域里，(fff) 的变化近似由内积 (∇f⋅Δθ\nabla f \cdot \Delta\theta∇f⋅Δθ) 决定。

1.1.2 直观含义（可操作的解释）

  • 梯度给出“函数增长最快的方向”：在所有单位方向向量 (vvv) 中，使 (f(θ+ϵv)f(\theta+\epsilon v)f(θ+ϵv)) 增大最快的方向就是梯度方向。

更形式地说，方向导数为：

Dvf(θ)=lim⁡ϵ→0f(θ+ϵv)−f(θ)ϵ=∇θf(θ)⊤vD_v f(\theta)=\lim_{\epsilon\to 0}\frac{f(\theta+\epsilon v)-f(\theta)}{\epsilon} = \nabla_\theta f(\theta)^\top v Dv​f(θ)=ϵ→0lim​ϵf(θ+ϵv)−f(θ)​=∇θ​f(θ)⊤v

当 (∣v∣=1|v|=1∣v∣=1) 时，最大值由柯西–施瓦茨不等式得到：

∇f(θ)⊤v≤∣∇f(θ)∣∣v∣=∣∇f(θ)∣\nabla f(\theta)^\top v \le |\nabla f(\theta)||v| = |\nabla f(\theta)| ∇f(θ)⊤v≤∣∇f(θ)∣∣v∣=∣∇f(θ)∣

等号当且仅当 (vvv) 与 (∇f\nabla f∇f) 同向成立。因此：

  • 沿着 (∇f\nabla f∇f) 走，(fff) 增加最快；
  • 沿着 (−∇f-\nabla f−∇f) 走，(fff) 减少最快。

这也解释了为什么“梯度”会自然地成为优化的方向信号：它提供了局部最陡方向。

1.2 梯度下降 vs 梯度上升

1.2.1 只是目标符号不同，不是方法本质不同

优化时你要么：

  • 最小化一个目标（例如损失）
  • 最大化一个目标（例如回报）

两者在数学上互为相反数。若最大化 (J(θ)J(\theta)J(θ))，等价于最小化 (−J(θ)-J(\theta)−J(θ))：

max⁡θJ(θ)⟺min⁡θ−J(θ)\max_\theta J(\theta) \quad\Longleftrightarrow\quad \min_\theta -J(\theta) θmax​J(θ)⟺θmin​−J(θ)

因此更新规则只是符号选择：

  • 最小化（梯度下降）：

θ←θ−α∇θL(θ)\theta \leftarrow \theta - \alpha \nabla_\theta L(\theta) θ←θ−α∇θ​L(θ)

  • 最大化（梯度上升）：

θ←θ+α∇θJ(θ)\theta \leftarrow \theta + \alpha \nabla_\theta J(\theta) θ←θ+α∇θ​J(θ)

这里 (α>0\alpha>0α>0) 是学习率。

关键结论：采用上升还是下降，完全由你的优化目标是“最大化”还是“最小化”决定。 算法本质仍是“沿着局部一阶近似的最陡方向移动”。

1.3 策略梯度定义

强化学习中，我们的“模型”不是直接输出一个数，而是输出一个策略（policy）：给定状态 (s)，输出对动作 (a) 的概率分布。

πθ(a∣s)\pi_\theta(a|s) πθ​(a∣s)

其中 (θ\thetaθ) 是策略参数（例如神经网络参数），(πθ(a∣s)\pi_\theta(a|s)πθ​(a∣s)) 是在状态 (sss) 选择动作 (aaa) 的概率。

强化学习的优化目标通常写成期望回报：

J(θ)=Eτ∼Pθ(τ)[R(τ)]J(\theta)=\mathbb{E}_{\tau\sim P_\theta(\tau)}[R(\tau)] J(θ)=Eτ∼Pθ​(τ)​[R(τ)]

  • (τ\tauτ) 是轨迹（trajectory），包含状态、动作序列：((s0,a0,s1,a1,… )(s_0,a_0,s_1,a_1,\dots)(s0​,a0​,s1​,a1​,…))
  • (Pθ(τ)P_\theta(\tau)Pθ​(τ)) 是在策略 (πθ\pi_\thetaπθ​) 下产生轨迹的概率分布
  • (R(τ)R(\tau)R(τ)) 是该轨迹的回报（return）

于是，“策略梯度（policy gradient）”指的就是：

∇θJ(θ)\nabla_\theta J(\theta) ∇θ​J(θ)

也就是：对策略参数求导，让期望回报变大。

和监督学习最重要的不同点在于： 这里的目标函数是一个对“轨迹分布”取期望的量，(θ\thetaθ) 影响的是 (Pθ(τ)P_\theta(\tau)Pθ​(τ))（即采样分布），而不是直接给出一个可微的标量输出供你对齐标签。

在 LLM 的 RLHF 里，策略就是语言模型的 token 分布：

πθ(yt∣y<t,x)\pi_\theta(y_t|y_{<t},x) πθ​(yt​∣y<t​,x)

轨迹是完整生成序列 (y=(y1,…,yT)y=(y_1,\dots,y_T)y=(y1​,…,yT​))，回报是 reward model 对整段输出的评分 (r(y)r(y)r(y))。形式完全一致：

J(θ)=Ey∼πθ(⋅∣x)[r(y)]J(\theta)=\mathbb{E}_{y\sim \pi_\theta(\cdot|x)}[r(y)] J(θ)=Ey∼πθ​(⋅∣x)​[r(y)]

1.4 监督学习 vs 强化学习的差异

这一节只强调最核心、与后文推导直接相关的差异：优化对象、数据生成机制、以及梯度信号的来源。

1.4.1 优化对象不同：拟合标签 vs 优化期望回报

  • 监督学习：数据集提供“正确答案/标签”，目标是最小化预测误差

min⁡θE(x,y)∼D[ℓ(fθ(x),y)]\min_\theta \mathbb{E}_{(x,y)\sim D}\big[\ell(f_\theta(x),y)\big] θmin​E(x,y)∼D​[ℓ(fθ​(x),y)]

  • 强化学习：没有逐步标签，目标是最大化交互产生的期望回报

max⁡θEτ∼Pθ[R(τ)]\max_\theta \mathbb{E}_{\tau\sim P_\theta}[R(\tau)] θmax​Eτ∼Pθ​​[R(τ)]

监督学习里的误差信号来自“标签差”，强化学习里来自“回报好坏”。

1.4.2 数据生成机制不同：离线固定数据 vs 在线采样轨迹

  • 监督学习：样本 ((x,y)) 在数据集里是固定的，你读取它们并计算梯度
  • 强化学习：样本（轨迹）需要由当前策略去环境中生成/采样出来

这一点决定了：强化学习里你讨论的不是“对固定样本的梯度”，而是“对随机采样轨迹的梯度估计器”（这也是为什么后文必须谈方差、baseline、PPO 等稳定化技术）。

1.4.3 可微性链路不同：计算图可见 vs 黑箱环境/采样节点

  • 监督学习的计算图通常是：

    θ→fθ(x)→ℓ(⋅,y)\theta \rightarrow f_\theta(x) \rightarrow \ell(\cdot,y) θ→fθ​(x)→ℓ(⋅,y)

    全部在模型内部，端到端可微。

  • 强化学习的交互链路通常是：

    θ→πθ(a∣s)→采样 a→环境→R\theta \rightarrow \pi_\theta(a|s) \rightarrow \text{采样 }a \rightarrow \text{环境} \rightarrow R θ→πθ​(a∣s)→采样 a→环境→R

    其中“采样动作”和“环境转移”通常不可微或不可得解析形式。

因此强化学习要用“对分布求导”的方式（后文的 log-likelihood trick / score function）把梯度写成可采样估计的形式。

1.4.4 信号结构不同：稠密监督 vs 延迟回报（信用分配）

监督学习通常每个样本都有直接误差信号；强化学习常见设置是回报在轨迹末端或稀疏出现，导致信用分配（credit assignment）更难、方差更大。LLM 的 RLHF 就属于典型的“序列末端给分”的结构：reward 对整段生成打分，但梯度要分摊到每个 token 的采样决策上。

本节小结

  • 梯度是局部一阶变化率向量；下降/上升仅由目标的最小化/最大化决定。
  • 策略梯度是强化学习中对 (J(θ)=E[R]J(\theta)=\mathbb{E}[R]J(θ)=E[R]) 求导，其核心是：(θ\thetaθ) 改变的是采样分布而非直接标签误差。
  • 监督学习与强化学习的关键差异在于数据生成与可微链路：前者“固定数据 + 直接误差”，后者“采样轨迹 + 分布求导”，这自然引出后文的 log-likelihood trick 与方差控制工具。

2. 从第一性原理推导策略梯度

本节要解决两件事：

  1. 数学上：从 (J(θ)=E[R]J(\theta)=\mathbb{E}[R]J(θ)=E[R]) 出发推导出可采样估计的梯度形式
  2. 概念上：澄清“不是不能对 (E[R]\mathbb{E}[R]E[R]) 求导，而是不能解析枚举求导；需要通过采样估计”

2.1 数学推导（含中间 trick）

2.1.1 目标函数：期望回报

定义轨迹（trajectory）：

τ=(s0,a0,s1,a1,…,sT)\tau=(s_0,a_0,s_1,a_1,\dots,s_T) τ=(s0​,a0​,s1​,a1​,…,sT​)

策略为 (πθ(a∣s)\pi_\theta(a|s)πθ​(a∣s))，轨迹分布为：

Pθ(τ)=ρ(s0)∏t=0T−1πθ(at∣st) P(st+1∣st,at)P_\theta(\tau)=\rho(s_0)\prod_{t=0}^{T-1}\pi_\theta(a_t|s_t)\,P(s_{t+1}|s_t,a_t) Pθ​(τ)=ρ(s0​)t=0∏T−1​πθ​(at​∣st​)P(st+1​∣st​,at​)

回报（return）记为 (R(τ)R(\tau)R(τ))。优化目标：

J(θ)=Eτ∼Pθ[R(τ)]=∑τPθ(τ)R(τ)J(\theta)=\mathbb{E}_{\tau\sim P_\theta}[R(\tau)] =\sum_{\tau}P_\theta(\tau)R(\tau) J(θ)=Eτ∼Pθ​​[R(τ)]=τ∑​Pθ​(τ)R(τ)

这里写成求和是离散轨迹情形；连续情形就是积分，推导一致。

2.1.2 对期望求导（第一性原理）

直接对 (θ\thetaθ) 求导：

∇θJ(θ)=∑τ∇θPθ(τ) R(τ)\nabla_\theta J(\theta)=\sum_\tau \nabla_\theta P_\theta(\tau)\,R(\tau) ∇θ​J(θ)=τ∑​∇θ​Pθ​(τ)R(τ)

困难在于 (∇θPθ(τ)\nabla_\theta P_\theta(\tau)∇θ​Pθ​(τ)) 不好用：它不在“期望形式”，不利于采样估计。关键 trick 是把 (∇P\nabla P∇P) 变成 (P∇log⁡PP\nabla\log PP∇logP)。

对这里做几个阐述：

1) “不在期望形式”到底是什么意思？

你现在有：

∇θJ(θ)=∑τ∇θPθ(τ) R(τ)\nabla_\theta J(\theta)=\sum_\tau \nabla_\theta P_\theta(\tau)\,R(\tau) ∇θ​J(θ)=τ∑​∇θ​Pθ​(τ)R(τ)

这当然是对的。但问题是：在强化学习/LLM 里，(τ\tauτ) 的空间巨大（指数级），你无法枚举求和，所以你想做的是用采样来近似这个和：

∑τ(⋅)≈1N∑i=1N(⋅)∣τi\sum_\tau (\cdot) \approx \frac{1}{N}\sum_{i=1}^N (\cdot)\Big|_{\tau_i} τ∑​(⋅)≈N1​i=1∑N​(⋅)∣∣∣∣​τi​​

而 Monte Carlo 采样近似之所以成立，要求你写成下面这种结构：

∑τPθ(τ) f(τ)⇒Eτ∼Pθ[f(τ)]\sum_\tau P_\theta(\tau)\,f(\tau) \quad\Rightarrow\quad \mathbb{E}_{\tau\sim P_\theta}[f(\tau)] τ∑​Pθ​(τ)f(τ)⇒Eτ∼Pθ​​[f(τ)]

因为你采样的是 (τi∼Pθ\tau_i\sim P_\thetaτi​∼Pθ​)，也就是用 (PθP_\thetaPθ​) 当“权重”。

但你的式子是：

∑τ∇θPθ(τ) R(τ)\sum_\tau \nabla_\theta P_\theta(\tau)\,R(\tau) τ∑​∇θ​Pθ​(τ)R(τ)

这里的“权重”不是 (Pθ(τ)P_\theta(\tau)Pθ​(τ))，而是 (∇θPθ(τ)\nabla_\theta P_\theta(\tau)∇θ​Pθ​(τ))，它既：

  • 不是概率分布（不非负、和不为 1）
  • 也不是你能用“从某个分布采样”的方式自然得到的权重

所以它“不在期望形式”的真正含义是：

你没法直接把它解释成 “对某个可采样分布的期望”，因此无法用“采样均值”去构造一个自然的无偏估计器。

2) “不利于采样估计”具体卡在哪里？

假设你硬要用 (τi∼Pθ(τ)\tau_i\sim P_\theta(\tau)τi​∼Pθ​(τ)) 来采样（这是你唯一能采样的分布：用策略 rollout）。

那么你想要无偏估计：

∑τ∇θPθ(τ) R(τ)\sum_\tau \nabla_\theta P_\theta(\tau)\,R(\tau) τ∑​∇θ​Pθ​(τ)R(τ)

通常的套路是乘除以 (Pθ(τ)P_\theta(\tau)Pθ​(τ))：

∑τ∇θPθ(τ) R(τ)=∑τPθ(τ) ∇θPθ(τ)Pθ(τ)R(τ)⏟f(τ)=Eτ∼Pθ[∇θPθ(τ)Pθ(τ)R(τ)]\sum_\tau \nabla_\theta P_\theta(\tau)\,R(\tau) = \sum_\tau P_\theta(\tau)\,\underbrace{\frac{\nabla_\theta P_\theta(\tau)}{P_\theta(\tau)}R(\tau)}_{f(\tau)} = \mathbb{E}_{\tau\sim P_\theta}\left[\frac{\nabla_\theta P_\theta(\tau)}{P_\theta(\tau)}R(\tau)\right] τ∑​∇θ​Pθ​(τ)R(τ)=τ∑​Pθ​(τ)f(τ)Pθ​(τ)∇θ​Pθ​(τ)​R(τ)​​=Eτ∼Pθ​​[Pθ​(τ)∇θ​Pθ​(τ)​R(τ)]

你会发现：这一步本质上就是 log-likelihood trick，因为：

∇θPθ(τ)Pθ(τ)=∇θlog⁡Pθ(τ)\frac{\nabla_\theta P_\theta(\tau)}{P_\theta(\tau)}=\nabla_\theta \log P_\theta(\tau) Pθ​(τ)∇θ​Pθ​(τ)​=∇θ​logPθ​(τ)

所以“不利于采样估计”的含义是：

  • 如果你不做这步变形，你不知道该用什么分布采样，也不知道单样本该怎么计算贡献
  • 做了这步变形后，你就能用 (τ∼Pθ\tau\sim P_\thetaτ∼Pθ​) 采样，并且每条样本只需要算 (R(τ)R(\tau)R(τ)) 和 (∇log⁡Pθ(τ)\nabla \log P_\theta(\tau)∇logPθ​(τ)) 就能得到无偏估计

3) “关键 trick 的逻辑”其实就是把“导数权重”换成“概率权重”

你可以把 log-likelihood trick 解释成一句非常具体的话：

我们只能从 (Pθ(τ)P_\theta(\tau)Pθ​(τ)) 这个概率分布采样， 所以必须把求和写成 (∑τPθ(τ)×(某个函数)\sum_\tau P_\theta(\tau)\times (\text{某个函数})∑τ​Pθ​(τ)×(某个函数))， 才能用 Monte Carlo 平均来估计它。

而

∇θPθ(τ)=Pθ(τ)∇θlog⁡Pθ(τ)\nabla_\theta P_\theta(\tau)=P_\theta(\tau)\nabla_\theta\log P_\theta(\tau) ∇θ​Pθ​(τ)=Pθ​(τ)∇θ​logPθ​(τ)

正好完成这件事：把“不可采样的权重 (∇P\nabla P∇P)”变成“可采样的权重 (PPP)”乘上一个可计算的校正项 (∇log⁡P\nabla\log P∇logP)。

2.1.3 Log-likelihood trick / Score function identity

对任意可微且 (Pθ(τ)>0P_\theta(\tau)>0Pθ​(τ)>0) 的概率，有：

∇θlog⁡Pθ(τ)=1Pθ(τ)∇θPθ(τ)⇒∇θPθ(τ)=Pθ(τ)∇θlog⁡Pθ(τ)\nabla_\theta \log P_\theta(\tau) =\frac{1}{P_\theta(\tau)}\nabla_\theta P_\theta(\tau) \quad\Rightarrow\quad \nabla_\theta P_\theta(\tau)=P_\theta(\tau)\nabla_\theta \log P_\theta(\tau) ∇θ​logPθ​(τ)=Pθ​(τ)1​∇θ​Pθ​(τ)⇒∇θ​Pθ​(τ)=Pθ​(τ)∇θ​logPθ​(τ)

将其代回：

∇θJ(θ)=∑τPθ(τ) R(τ) ∇θlog⁡Pθ(τ)=Eτ∼Pθ[R(τ)∇θlog⁡Pθ(τ)]\nabla_\theta J(\theta) =\sum_\tau P_\theta(\tau)\,R(\tau)\,\nabla_\theta \log P_\theta(\tau) =\mathbb{E}_{\tau\sim P_\theta}\big[R(\tau)\nabla_\theta \log P_\theta(\tau)\big] ∇θ​J(θ)=τ∑​Pθ​(τ)R(τ)∇θ​logPθ​(τ)=Eτ∼Pθ​​[R(τ)∇θ​logPθ​(τ)]

这一步非常关键：我们得到一个“期望形式”的梯度，因此可以 Monte Carlo 采样估计。

2.1.4 将轨迹对数概率拆开（环境项消失）

把 (log⁡Pθ(τ)\log P_\theta(\tau)logPθ​(τ)) 展开：

log⁡Pθ(τ)=log⁡ρ(s0)+∑t=0T−1log⁡πθ(at∣st)+∑t=0T−1log⁡P(st+1∣st,at)\log P_\theta(\tau) =\log\rho(s_0)+\sum_{t=0}^{T-1}\log\pi_\theta(a_t|s_t)+\sum_{t=0}^{T-1}\log P(s_{t+1}|s_t,a_t) logPθ​(τ)=logρ(s0​)+t=0∑T−1​logπθ​(at​∣st​)+t=0∑T−1​logP(st+1​∣st​,at​)

注意：环境转移 (P(st+1∣st,at)P(s_{t+1}|s_t,a_t)P(st+1​∣st​,at​)) 与初始分布 (ρ(s0)\rho(s_0)ρ(s0​)) 不依赖 (θ\thetaθ)（(θ\thetaθ) 只参数化策略），因此

∇θlog⁡Pθ(τ)=∑t=0T−1∇θlog⁡πθ(at∣st)\nabla_\theta \log P_\theta(\tau)=\sum_{t=0}^{T-1}\nabla_\theta \log\pi_\theta(a_t|s_t) ∇θ​logPθ​(τ)=t=0∑T−1​∇θ​logπθ​(at​∣st​)

代回得到经典策略梯度定理（蒙特卡洛形式）：

∇θJ(θ)=Eτ∼Pθ[R(τ)∑t=0T−1∇θlog⁡πθ(at∣st)]\boxed{ \nabla_\theta J(\theta)= \mathbb{E}_{\tau\sim P_\theta} \left[ R(\tau)\sum_{t=0}^{T-1}\nabla_\theta \log\pi_\theta(a_t|s_t) \right] } ∇θ​J(θ)=Eτ∼Pθ​​[R(τ)t=0∑T−1​∇θ​logπθ​(at​∣st​)]​

这也解释了一个常见误解： 强化学习里我们并不是“对环境求导”，而是把梯度写成只包含 (∇log⁡π\nabla\log\pi∇logπ) 的形式，环境只通过采样产生 (R(τ)R(\tau)R(τ)) 影响更新。

2.1.5 采样估计（有限 batch）

用 (NNN) 条轨迹采样 (τi∼Pθ\tau_i\sim P_\thetaτi​∼Pθ​)：

∇θJ^=1N∑i=1NR(τi)∑t∇θlog⁡πθ(ai,t∣si,t)\widehat{\nabla_\theta J} =\frac{1}{N}\sum_{i=1}^N R(\tau_i)\sum_{t}\nabla_\theta \log\pi_\theta(a_{i,t}|s_{i,t}) ∇θ​J​=N1​i=1∑N​R(τi​)t∑​∇θ​logπθ​(ai,t​∣si,t​)

这就是策略梯度在工程里训练时真正用到的“梯度估计器”。它是随机变量，因此有方差（第 3 节会系统展开）。

2.2 数值案例说明：离散 2 动作系统，枚举所有轨迹验证一致性

为了让“求导形式 = 期望形式”变成可检查的事实，我们构造一个能完全枚举的 toy。

场景设定（一步轨迹）

  • 单状态 (sss)，两动作 (A,BA,BA,B)
  • 策略参数 (θ\thetaθ) 直接表示 (p=πθ(B)p=\pi_\theta(B)p=πθ​(B))，因此 (πθ(A)=1−p\pi_\theta(A)=1-pπθ​(A)=1−p)
  • 回报：若选 (BBB) 得 10 分，若选 (AAA) 得 0 分

则目标函数：

J(p)=E[R]=p⋅10+(1−p)⋅0=10pJ(p)=\mathbb{E}[R]=p\cdot 10+(1-p)\cdot 0 = 10p J(p)=E[R]=p⋅10+(1−p)⋅0=10p

解析求导：

dJdp=10\frac{dJ}{dp}=10 dpdJ​=10

用“期望形式策略梯度”再算一遍

轨迹只有两种：(τ=A\tau=Aτ=A) 或 (τ=B\tau=Bτ=B)

  • (P(τ=B)=pP(\tau=B)=pP(τ=B)=p)，(R(B)=10R(B)=10R(B)=10)
  • (P(τ=A)=1−pP(\tau=A)=1-pP(τ=A)=1−p)，(R(A)=0R(A)=0R(A)=0)

并且：

ddplog⁡P(τ=B)=ddplog⁡p=1p\frac{d}{dp}\log P(\tau=B)=\frac{d}{dp}\log p=\frac{1}{p} dpd​logP(τ=B)=dpd​logp=p1​

ddplog⁡P(τ=A)=ddplog⁡(1−p)=−11−p\frac{d}{dp}\log P(\tau=A)=\frac{d}{dp}\log(1-p)=-\frac{1}{1-p} dpd​logP(τ=A)=dpd​log(1−p)=−1−p1​

期望形式：

dJdp=E[R(τ)ddplog⁡P(τ)]=p⋅10⋅1p+(1−p)⋅0⋅(−11−p)=10\frac{dJ}{dp} =\mathbb{E}\big[R(\tau)\frac{d}{dp}\log P(\tau)\big] =p\cdot 10\cdot \frac{1}{p} + (1-p)\cdot 0\cdot\left(-\frac{1}{1-p}\right) =10 dpdJ​=E[R(τ)dpd​logP(τ)]=p⋅10⋅p1​+(1−p)⋅0⋅(−1−p1​)=10

与解析求导完全一致。

这例子回答了我们对话里的关键争议： 并不是“不能对 (E[R]\mathbb{E}[R]E[R]) 求导”，而是大问题里轨迹空间巨大、无法枚举求和，只能用采样估计期望。
