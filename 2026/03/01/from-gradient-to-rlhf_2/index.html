从梯度到 RLHF：策略梯度的数学本质、方差来源与工程控制（Part II）

3. 策略梯度对于动作概率的影响

本节解决三个问题：

  1. 数学上：策略梯度更新如何导致“被采样动作概率上升/下降”
  2. 数值上：用具体数字展示概率变化
  3. 统计上：解释“策略梯度方差为什么大”，并落到 LLM/RLHF + PPO/KL

3.1 数学推导：动作概率变化与策略梯度的关系

为了严谨起见，先写出带期望的策略梯度形式。策略梯度定理给出：

∇θJ(θ)=Es∼dπθ, a∼πθ(⋅∣s)[Aπθ(s,a) ∇θlog⁡πθ(a∣s)]\nabla_\theta J(\theta) = \mathbb{E}_{s\sim d^{\pi_\theta},\,a\sim \pi_\theta(\cdot|s)} \big[ A^{\pi_\theta}(s,a)\,\nabla_\theta \log \pi_\theta(a|s) \big] ∇θ​J(θ)=Es∼dπθ​,a∼πθ​(⋅∣s)​[Aπθ​(s,a)∇θ​logπθ​(a∣s)]

其中：

  • (dπθ(s)d^{\pi_\theta}(s)dπθ​(s)) 是当前策略诱导的状态分布
  • (Aπθ(s,a)A^{\pi_\theta}(s,a)Aπθ​(s,a)) 是 advantage（可以用 (Qπ−VπQ^\pi-V^\piQπ−Vπ) 或采样回报减 baseline 估计）

训练时我们无法计算这个期望，只能通过采样构造一个无偏随机梯度估计器：

∇θJ^=A^(s,a) ∇θlog⁡πθ(a∣s),(s,a)∼πθ\widehat{\nabla_\theta J} = \hat A(s,a)\,\nabla_\theta \log \pi_\theta(a|s) ,\quad (s,a)\sim \pi_\theta ∇θ​J​=A^(s,a)∇θ​logπθ​(a∣s),(s,a)∼πθ​

然后进行一次随机梯度上升更新：

θ←θ+α ∇θJ^=θ+α A^(s,a) ∇θlog⁡πθ(a∣s)\theta \leftarrow \theta + \alpha\, \widehat{\nabla_\theta J} = \theta + \alpha\, \hat A(s,a)\,\nabla_\theta \log \pi_\theta(a|s) θ←θ+α∇θ​J​=θ+αA^(s,a)∇θ​logπθ​(a∣s)

下面的“概率变化分析”，指的是单次采样更新对概率的局部影响方向。

在 logit 空间分析概率变化

为了看“概率怎么变”，最直接方式是分析 softmax 的 logit。

设策略为：

πθ(i∣s)=ezi∑jezj\pi_\theta(i|s)=\frac{e^{z_i}}{\sum_j e^{z_j}} πθ​(i∣s)=∑j​ezj​ezi​​

其中 (zi=zi(θ,s)z_i=z_i(\theta,s)zi​=zi​(θ,s)) 是动作 (iii) 的 logit。

为了研究更新方向，固定某个状态 (sss)，把 (zzz) 当作参数（暂时忽略 (θ\thetaθ) 到 (zzz) 的链式关系），考察：

∂∂zilog⁡π(a∣s)={1−π(a∣s)i=a−π(i∣s)i≠a\frac{\partial}{\partial z_i}\log \pi(a|s) = \begin{cases} 1-\pi(a|s) & i=a\\ -\pi(i|s) & i\neq a \end{cases} ∂zi​∂​logπ(a∣s)={1−π(a∣s)−π(i∣s)​i=ai=a​

因此，在一次采样到动作 (aaa) 并得到 advantage 估计 (A^(s,a)\hat A(s,a)A^(s,a)) 后，对应的 logit 更新方向为：

  • 选中动作 (aaa)：

Δza=α A^(s,a) (1−π(a∣s))\Delta z_a=\alpha\,\hat A(s,a)\,(1-\pi(a|s)) Δza​=αA^(s,a)(1−π(a∣s))

  • 其他动作 (i≠ai\neq ai=a)：

Δzi=−α A^(s,a) π(i∣s)\Delta z_i=-\alpha\,\hat A(s,a)\,\pi(i|s) Δzi​=−αA^(s,a)π(i∣s)

（这里忽略了从 (θ\thetaθ) 到 (zzz) 的链式法则系数，只保留方向与比例结构。）

旁注：

  • 如果你用的是 (RRR)（没有 baseline / advantage），并且 (R≥0R\ge 0R≥0) 总是成立，那么每次采样都会“强化”采样到的动作（提高其概率），这点没错。
  • 你可能会进一步问：既然这里的增减看起来与 (AAA) 的正负有关，那在没有 (AAA) 的算法里（直接用 (RRR)），而且通常 (R≥0R\ge 0R≥0)，是不是就意味着“无论采到好动作还是坏动作，只要采样到就一定增加它的概率”？
  • 但这并不意味着“学不到坏动作会被压下去”。原因是：softmax 的更新是耦合的，你提高某个动作的概率，会压低其他动作的概率；而“好动作被强化得更快”，“坏动作虽然也被强化但强化得更慢”，长期看坏动作会被挤出去。
  • 另外，实际算法里通常会用 baseline/advantage 的一个重要原因就是：如果 reward 永远非负，你永远只能做正向强化，方差大、学习慢、并且很容易出现错误 credit assignment。baseline 让你得到“相对好坏”（正负）信号，更新更有效。

下面将逻辑分 4 层讲透，并给一个数值例子证明“即便 (R≥0R\ge 0R≥0)，坏动作仍会下降”。

1) 没有 advantage 时，更新确实由 (RRR) 的正负决定

没有 baseline/advantage 的 REINFORCE 单步更新是：

θ←θ+α R ∇θlog⁡πθ(a∣s)\theta \leftarrow \theta + \alpha\, R\,\nabla_\theta \log \pi_\theta(a|s) θ←θ+αR∇θ​logπθ​(a∣s)

在 logit 空间（固定 (sss)，把 (zzz) 当参数）：

  • 选中动作 (aaa)：

Δza=αR(1−π(a∣s))\Delta z_a = \alpha R(1-\pi(a|s)) Δza​=αR(1−π(a∣s))

  • 其他动作 (i≠ai\neq ai=a)：

Δzi=−αR π(i∣s)\Delta z_i = -\alpha R\,\pi(i|s) Δzi​=−αRπ(i∣s)

所以如果 (R>0R>0R>0)：

  • 选中动作的 logit 上升
  • 其他动作的 logit 下降

这意味着：每次采样到哪个动作，就把它往上推。

你这一点判断完全正确。

2) 那“坏动作”怎么会被压下去？关键在于“好动作推得更快”

虽然每次采样到的动作都会被推上去，但不同动作被采样到的频率、以及推上去的力度（由 (RRR) 决定）是不一样的。

直观地讲：

  • 好动作采样到时 (RRR) 大 ⇒ 每次推得猛
  • 坏动作采样到时 (RRR) 小（虽然仍 ≥0\ge 0≥0）⇒ 每次推得弱
  • 同时好动作在推高后概率会变大 ⇒ 被采样到更频繁 ⇒ 形成正反馈
  • 坏动作概率相对减少 ⇒ 被采样到更少 ⇒ 被推高的机会更少

因此即便所有 (R≥0R\ge 0R≥0)，你依然会得到“相对选择”：好动作越来越占概率质量，坏动作被挤压到很小。

3) 一个具体数值例子：(R≥0R\ge 0R≥0) 但坏动作概率仍然下降

单状态，两动作 (A,BA,BA,B)：

  • 若选 (AAA)：奖励 (RA=10R_A = 10RA​=10)
  • 若选 (BBB)：奖励 (RB=1R_B = 1RB​=1)（始终正的“坏动作”）

初始概率设为：

π(A)=π(B)=0.5\pi(A)=\pi(B)=0.5 π(A)=π(B)=0.5

学习率 (α=0.1\alpha=0.1α=0.1)。

第 1 次采样：假设采到 B（坏动作），(R=1R=1R=1)

logit 更新（用上面公式）：

  • (ΔzB=0.1×1×(1−0.5)=0.05\Delta z_B = 0.1\times 1 \times (1-0.5)=0.05ΔzB​=0.1×1×(1−0.5)=0.05)
  • (ΔzA=−0.1×1×0.5=−0.05\Delta z_A = -0.1\times 1 \times 0.5=-0.05ΔzA​=−0.1×1×0.5=−0.05)

这次确实“强化”了 B。

第 2 次采样：若之后采到 A（好动作），(R=10R=10R=10)

此时 (AAA) 的更新力度是 B 的 10 倍：

  • (ΔzA≈0.1×10×(1−π(A))\Delta z_A \approx 0.1\times 10 \times (1-\pi(A))ΔzA​≈0.1×10×(1−π(A)))（接近 0.5 时大约 0.5）
  • 其它动作被压下去的幅度也更大

结果通常是：只要 A 被采到几次，A 的概率会迅速接近 1，B 接近 0。

也就是说：坏动作不是因为“负奖励”才下降，而是因为“好动作得到更大的正奖励 + 更频繁被采样”，从而把概率质量挤走了。

4) 那 advantage/baseline 的价值是什么？——不是“让坏动作才会下降”，而是“让更新更高效、更低方差”

你发现了一个事实：如果 (R≥0R\ge 0R≥0)，更新方向只会“强化采样到的动作”。那为什么大家还要引入 advantage（允许为负）？

核心原因有两个：

(1) 把“绝对好坏”变成“相对好坏”，提高学习效率

用 (RRR) 时，你在学的是：

  • “只要有奖励就加强”

但用 advantage：

A(s,a)=R−b(s)A(s,a)=R-b(s) A(s,a)=R−b(s)

你学的是：

  • “比平均水平好就加强”
  • “比平均水平差就削弱”

这在 同一状态下区分动作时更直接，学习更快。

(2) 降低方差（这是最重要的工程原因）

即便 (R≥0R\ge 0R≥0)，你仍然会有很大噪声：同一个动作在不同轨迹里回报可能波动很大。baseline 把回报做中心化，显著降低估计器方差，同时不改变期望梯度。

5) 关于提到的“那不就总是提升采样到的动作吗？”——补一句更精确的话

更精确的说法是：

  • 在 没有 baseline 且 (R≥0R\ge 0R≥0) 的情况下，每次采样确实会提高采样到动作的 logit（局部方向为正）。
  • 但从期望意义上看，策略会更偏向“带来更高期望回报的动作”，因为它们带来的更新幅度更大、发生更频繁。

而 baseline/advantage 的作用是把这种“靠长期挤压形成的相对选择”变成“单步就有相对方向”的更新，提高样本效率并降低方差。

没有 advantage 时也能学到：只要好动作回报更高，它会被更强更频繁地强化，最终挤压坏动作的概率。 但 advantage 能更快更稳：把“绝对回报”变成“相对回报”，产生正负信号并降低方差。

严格结论

  1. 若 (A^(s,a)>0\hat A(s,a)>0A^(s,a)>0)（该动作优于基线）：

    • (Δza>0\Delta z_a>0Δza​>0)，选中动作 logit 上升
    • 其余动作 logit 下降
    • 因 softmax 归一化，选中动作概率上升

  2. 若 (A^(s,a)<0\hat A(s,a)<0A^(s,a)<0)（该动作劣于基线）：

    • (Δza<0\Delta z_a<0Δza​<0)，选中动作 logit 下降
    • 其余动作 logit 上升
    • 选中动作概率下降

  3. 变化幅度与 (A^(s,a)\hat A(s,a)A^(s,a)) 成正比

    • (A^\hat AA^) 越大，推动越强
    • (A^\hat AA^) 越接近 0，更新越小
    • 不存在“阈值”——只要 (A^≠0\hat A\neq 0A^=0)，概率就会发生连续变化

在我们之前的讨论中有一个关键问题： “奖励高到底多高才会提升概率？奖励低到底多低才会降低？”

数学上的答案非常直接： 由 advantage 的大小线性决定，没有阈值机制。

策略梯度不是分类决策规则，而是一个连续的梯度系统。

二、这个假设是否合理？

关键是：你在这里是在分析方差的数量级，而不是在说真实训练中“均值为 0”。在 REINFORCE 中

g=R∑t=1Tutg=R\sum_{t=1}^{T}u_t g=Rt=1∑T​ut​

这里用 E[ut]=0\mathbb{E}[u_t]=0E[ut​]=0 更像是一种“中心化近似”：只想把注意力放在 score term 本身的内在随机性上，从而得到 Var(∑tut)\mathrm{Var}(\sum_t u_t)Var(∑t​ut​) 随长度 TTT 线性增长的量级结论。

3.3.3 “末尾 reward”导致的 SNR 随 TTT 下降

当 reward 只在末尾给（典型 RLHF：整段文本一个分数），但梯度要分摊到每个 token：

  • 只有少数 token 真正对 reward 有“可预测贡献”
  • 其余 token 的 utu_tut​ 对最终 RRR 的相关性弱

形式化地，如果

E[g]=μ,Var(g)≈Tσ2\mathbb{E}[g]=\mu \quad,\quad \mathrm{Var}(g)\approx T\sigma^2 E[g]=μ,Var(g)≈Tσ2

则信噪比：

SNR=∣E[g]∣2Var(g)∝1T\mathrm{SNR}=\frac{|\mathbb{E}[g]|^2}{\mathrm{Var}(g)} \propto \frac{1}{T} SNR=Var(g)∣E[g]∣2​∝T1​

长度越长，单位样本的有效学习信号越稀释，训练就越依赖大 batch、critic、GAE 等降方差手段。

额外说明：SNR 高低对模型训练会产生什么影响？

这个问题本质上是：

梯度估计的信噪比（SNR）如何影响优化动力学？

我们分三层讲清楚：

  1. SNR 在随机优化中的数学含义
  2. SNR 下降对训练动态的直接影响
  3. 在“末尾 reward”场景下为什么问题会特别严重
一、SNR 在梯度估计里的精确定义

设随机梯度（随机估计量）：

g=∇θJ(θ)g = \nabla_\theta J(\theta) g=∇θ​J(θ)

其期望为

定义

SNR=∣μ∣2Tr(Σ)\mathrm{SNR} = \frac{|\mu|^2}{\mathrm{Tr}(\Sigma)} SNR=Tr(Σ)∣μ∣2​

它衡量的是：

优化更新是：

θk+1=θk+ηg\theta_{k+1} = \theta_k + \eta g θk+1​=θk​+ηg

其中：

  • μ\muμ = 真正有方向性的改进信号
  • ε\varepsilonε = 纯噪声
二、SNR 高低对训练的影响
情况 1：SNR 高（∣μ∣|\mu|∣μ∣ 远大于噪声尺度）
  • 更新方向稳定
  • 每步都朝近似一致方向移动
  • 收敛速度快
  • 不容易震荡
  • 不需要极大 batch

等价于：

“梯度指向明确”

情况 2：SNR 低（噪声远大于有效信号）

θk+1=θk+η(μ+ε)\theta_{k+1} = \theta_k + \eta(\mu + \varepsilon) θk+1​=θk​+η(μ+ε)

问题：

  1. 方向抖动严重
  2. 有效更新被随机波动淹没
  3. 需要更小学习率
  4. 需要更大 batch
  5. 收敛速度显著变慢

E[ΔJ]≈η ∣μ∣2−c η2 Tr(Σ)\mathbb{E}[\Delta J] \approx \eta\,|\mu|^2 - c\,\eta^2\,\mathrm{Tr}(\Sigma) E[ΔJ]≈η∣μ∣2−cη2Tr(Σ)

  • 二阶噪声项变大
  • 实际收益可能被抵消

这就是：

低 SNR ⇒\Rightarrow⇒ 优化效率下降

三、在“末尾 reward”场景下为什么问题特别严重？

你上面的推导是：

Var(g)≈Tσ2\mathrm{Var}(g) \approx T\sigma^2 Var(g)≈Tσ2

E[g]=μ(通常不随 T 强正比增长)\mathbb{E}[g] = \mu \quad (\text{通常不随 } T \text{ 强正比增长}) E[g]=μ(通常不随 T 强正比增长)

  • T=100T=100T=100 个 token

1：训练更不稳定

长回答会导致：

  • 梯度波动巨大
  • 同样 prompt 多次采样方向不同
  • loss 曲线抖动
2：学习速度变慢

因为

有效梯度强度∝1T\text{有效梯度强度} \propto \frac{1}{T} 有效梯度强度∝T1​

长文本样本：

  • 单样本贡献变小
  • 需要更多样本才能抵消噪声
3：更依赖大 batch

Batch 平均后：

Var(gbatch)=1B Var(g)\mathrm{Var}(g_{\text{batch}}) = \frac{1}{B}\,\mathrm{Var}(g) Var(gbatch​)=B1​Var(g)

所以：

这解释了：长序列 RL 训练必须增大 batch 才更稳定。

如果使用

其中

At=R−V(st)A_t = R - V(s_t) At​=R−V(st​)

则：

  • 不相关 token 的优势接近 0
  • 方差显著下降

等价于：

把噪声 token 的权重压低

五、极端情况：SNR 太低会发生什么？

当

∣μ∣2≪Tr(Σ)|\mu|^2 \ll \mathrm{Tr}(\Sigma) ∣μ∣2≪Tr(Σ)

可能出现：

1：学习停滞

更新方向接近随机游走，模型性能不提升。

2：KL 爆炸

因为：

  • 噪声更新导致 policy 在错误方向大幅移动
  • PPO clip 频繁触发
3：“奖励黑客”更容易发生

低 SNR 时：

  • 优化更像是高方差探索
  • 更容易偶然碰到 reward hacking 局部模式
六、一个直观类比

想象：

  • μ\muμ = 你想走的方向
  • ε\varepsilonε = 每秒刮来的随机风

如果风速比你走路速度大，你基本在原地打转。这就是低 SNR。

在“末尾 reward”+ 长序列情况下

一句话概括：

低 SNR 不是“训练难一点”，而是直接决定优化是否可行。

3.4 LLM 训练中方差的来源（随机变量 + 三大“方差大头”）

这一节要非常明确：方差来自哪些随机变量，以及为何在 LLM 中尤其严重。

3.4.1 明确随机变量

在 RLHF/RLAIF 中，至少包括：

  • token：yt∼πθ(⋅∣y<t,x)y_t \sim \pi_\theta(\cdot\mid y_{<t},x)yt​∼πθ​(⋅∣y<t​,x)
  • 序列长度 TTT（可能是随机或由 stopping 决定）
  • 回报：R(y)=r(y)R(y)=r(y)R(y)=r(y)（reward model 给定 yyy 通常确定，但因 yyy 随机，RRR 作为随机变量仍波动）
  • PPO 里还会引入 importance ratio：r(y)=πθ(y)πold(y)r(y)=\frac{\pi_\theta(y)}{\pi_{\text{old}}(y)}r(y)=πold​(y)πθ​(y)​（见 3.5）

3.4.2 token 累积：∑tut\sum_t u_t∑t​ut​ 的方差随 TTT 增长

如 3.3 所示：即便 RRR 不波动，S=∑tutS=\sum_t u_tS=∑t​ut​ 的方差也随长度增大。

3.4.3 高维放大：梯度向量的“能量”随维度增长

设参数维度为 ddd。若每一维梯度方差约为 Tσ2T\sigma^2Tσ2，则平方范数的期望规模是：

E∣g∣2≈d⋅Tσ2\mathbb{E}|g|^2 \approx d\cdot T\sigma^2 E∣g∣2≈d⋅Tσ2

当 ddd 是 10910^9109 量级、TTT 是几百量级时，即使每维方差看起来不大，整体梯度能量也非常巨大，训练对 clip、归一化、KL 约束极为敏感。

3.4.4 importance weight 爆炸：少量样本主导二阶矩

在 PPO/importance sampling 场景里，梯度会带上比率 rrr：

g∼r(y) A(y) ∇θlog⁡πθ(y)g \sim r(y)\,A(y)\,\nabla_\theta \log \pi_\theta(y) g∼r(y)A(y)∇θ​logπθ​(y)

如果 rrr 有重尾（少量样本 r≫1r\gg 1r≫1），则二阶矩

E[r2∣u∣2]\mathbb{E}[r^2 |u|^2] E[r2∣u∣2]

会被尾部样本主导，引发训练不稳定（这一点直接引出 KL 的意义）。

3.4.5 “实际训练”层面的直观例子（可写成经验描述）

同一个 prompt：

  • 有时采样到高 reward、但在旧策略下概率极低的回答（(r) 巨大）
  • 这条样本会让一次 update 的梯度尺度远大于其它样本
  • 于是出现：loss 抖动、KL 突增、reward 反复塌陷（典型 RLHF 不稳定现象）

3.5 PPO 中 KL 对方差的控制（importance sampling → 尾部控制 → trust region）

3.5.1 重要性采样（为什么会出现 ratio）

PPO 的常见形式用旧策略采样 (y∼πoldy\sim \pi_{\text{old}}y∼πold​)，再用新策略优化目标。会出现：

r(y)=πθ(y)πold(y)r(y)=\frac{\pi_\theta(y)}{\pi_{\text{old}}(y)} r(y)=πold​(y)πθ​(y)​

策略目标（简化写法）：

Ey∼πold[r(y)A(y)]\mathbb{E}_{y\sim\pi_{\text{old}}}\big[r(y)A(y)\big] Ey∼πold​​[r(y)A(y)]

梯度因此会被 (r(y)) 直接缩放，二阶矩对 (r2r^2r2) 敏感。

3.5.2 极端 ratio 的数值示例（方差爆炸的直观根源）

假设 batch 中：

  • 99% 样本 (r=1)
  • 1% 样本 (r=100)

则

E[r2]=0.99⋅12+0.01⋅1002=0.99+100=100.99\mathbb{E}[r^2]=0.99\cdot 1^2+0.01\cdot 100^2=0.99+100=100.99 E[r2]=0.99⋅12+0.01⋅1002=0.99+100=100.99

也就是说：哪怕极端样本只占 1%，它对二阶矩（从而对梯度方差）贡献几乎是全部。

3.5.3 KL 为什么能限制尾部（核心逻辑）

KL 正则（或 KL penalty）常写为：

β KL(πθ∣πref)=β Ey∼πθ[log⁡πθ(y)πref(y)]\beta\,\mathrm{KL}(\pi_\theta | \pi_{\text{ref}}) =\beta\,\mathbb{E}_{y\sim\pi_\theta}\left[\log\frac{\pi_\theta(y)}{\pi_{\text{ref}}(y)}\right] βKL(πθ​∣πref​)=βEy∼πθ​​[logπref​(y)πθ​(y)​]

当 KL 被压小，意味着 (log⁡πθπref\log\frac{\pi_\theta}{\pi_{\text{ref}}}logπref​πθ​​) 不能出现大量大偏移，从经验上会抑制 (r) 的重尾（尤其是把新策略推到旧策略极低概率区域时）。

PPO 的 clipping（常见实现）更直接地做了“硬截断”：

min⁡(rA,  clip(r,1−ϵ,1+ϵ)A)\min\big(rA,\;\mathrm{clip}(r,1-\epsilon,1+\epsilon)A\big) min(rA,clip(r,1−ϵ,1+ϵ)A)

它等价于显式限制 (r) 的极端值，从而控制二阶矩。

3.5.4 与 trust region 的联系（概念桥梁）

TRPO 的思想是直接约束策略更新不要超过一个 KL “信任域”：

max⁡θ E[rA]s.t.E[KL(πold∣πθ)]≤δ\max_\theta \ \mathbb{E}[rA] \quad \text{s.t.}\quad \mathbb{E}\big[\mathrm{KL}(\pi_{\text{old}}|\pi_\theta)\big]\le \delta θmax​ E[rA]s.t.E[KL(πold​∣πθ​)]≤δ

PPO 可以视为对 trust region 的可实现近似：通过 clipping/KL penalty，让策略分布在每次更新中不要偏移过大。偏移小 → ratio 不重尾 → 梯度二阶矩更可控 → 训练更稳定。

4. baseline 对期望和方差的影响

本节要完成三个目标：

  1. 证明：baseline 不改变策略梯度的期望（不引入偏差），条件是什么
  2. 说明：baseline 为什么能显著降方差（用全方差公式拆解）
  3. 给出：条件最优 baseline 与 batch-optimal baseline，并解释为什么 LLM 里不直接这么做

4.1 baseline 不改变期望（关键引理 + 结论）

用 baseline (b(s)) 替换回报 (R)：

g′=(R−b(s))∇θlog⁡πθ(a∣s)g'=(R-b(s))\nabla_\theta \log \pi_\theta(a|s) g′=(R−b(s))∇θ​logπθ​(a∣s)

要证明 (\mathbb{E}[g']=\mathbb{E}[g])，关键引理是：

Ea∼πθ(⋅∣s)[∇θlog⁡πθ(a∣s)]=0\boxed{ \mathbb{E}_{a\sim\pi_\theta(\cdot|s)}\big[\nabla_\theta \log \pi_\theta(a|s)\big]=0 } Ea∼πθ​(⋅∣s)​[∇θ​logπθ​(a∣s)]=0​

证明非常短：

E[∇log⁡π(a∣s)]=∑aπ(a∣s)∇log⁡π(a∣s)=∑a∇π(a∣s)=∇∑aπ(a∣s)=∇1=0\mathbb{E}[\nabla \log \pi(a|s)] =\sum_a \pi(a|s)\nabla \log \pi(a|s) =\sum_a \nabla \pi(a|s) =\nabla \sum_a \pi(a|s) =\nabla 1=0 E[∇logπ(a∣s)]=a∑​π(a∣s)∇logπ(a∣s)=a∑​∇π(a∣s)=∇a∑​π(a∣s)=∇1=0

因此，只要 baseline 不依赖动作 (a)（可以依赖状态 (s)、时间步 (t)、prompt (x)），就有：

E[b(s)∇log⁡π(a∣s)]=b(s)⋅0=0⇒E[g′]=E[g]\mathbb{E}[b(s)\nabla\log\pi(a|s)]=b(s)\cdot 0 = 0 \Rightarrow \mathbb{E}[g']=\mathbb{E}[g] E[b(s)∇logπ(a∣s)]=b(s)⋅0=0⇒E[g′]=E[g]

结论：baseline 不改变期望，只改变方差。 条件：baseline 不能依赖当下采样动作（否则会引入偏差风险）。

4.2 baseline 对方差的影响（全方差拆解视角）

用全方差公式（工具箱）：

Var(X)=E[Var(X∣Y)]+Var(E[X∣Y])\mathrm{Var}(X)=\mathbb{E}[\mathrm{Var}(X|Y)]+\mathrm{Var}(\mathbb{E}[X|Y]) Var(X)=E[Var(X∣Y)]+Var(E[X∣Y])

令

X=g=(R−b)u,u=∇θlog⁡π(a∣s),Y=aX=g=(R-b)u,\quad u=\nabla_\theta\log\pi(a|s),\quad Y=a X=g=(R−b)u,u=∇θ​logπ(a∣s),Y=a

则

Var(g)=E[Var(g∣a)]⏟回报噪声/环境随机性+Var(E[g∣a])⏟不同动作平均效果差异\mathrm{Var}(g) = \underbrace{\mathbb{E}[\mathrm{Var}(g|a)]}_{\text{回报噪声/环境随机性}} + \underbrace{\mathrm{Var}(\mathbb{E}[g|a])}_{\text{不同动作平均效果差异}} Var(g)=回报噪声/环境随机性E[Var(g∣a)]​​+不同动作平均效果差异Var(E[g∣a])​​

减去 baseline 不会改变第一项中的“条件方差”（因为减常数不改变 (R) 的条件方差），但可以显著缩小第二项，让不同动作的条件均值更接近，从而减少总体方差。

这解释了一个常见现象：即使 reward 本身非常 noisy，baseline 也只能消除其中“可解释的那部分结构性波动”，噪声底座仍然存在。

4.3 条件最优 baseline（公式 + 含义）

对标量参数（或把向量梯度用范数处理），令 u=∇θlog⁡πu=\nabla_\theta\log\piu=∇θ​logπ。在给定状态下，使 Var((R−b)u)\mathrm{Var}((R-b)u)Var((R−b)u) 最小的 baseline 满足：

b∗(s)=E[R u2∣s]E[u2∣s]\boxed{ b^*(s)=\frac{\mathbb{E}[R\,u^2|s]}{\mathbb{E}[u^2|s]} } b∗(s)=E[u2∣s]E[Ru2∣s]​​

这说明最优 baseline 不是简单的 E[R∣s]\mathbb{E}[R|s]E[R∣s]，而是按 u2u^2u2 加权——也就是更关注“对参数影响更大的样本”。

在工程中常用的 b(s)=V(s)≈E[R∣s]b(s)=V(s)\approx \mathbb{E}[R|s]b(s)=V(s)≈E[R∣s] 是一个强近似，优势在于易学、易泛化。

4.4 为什么 LLM 不直接用 batch-optimal baseline

既然存在最优 baseline，是否可以在每个 batch 用样本直接估它，让方差最小？

4.4.1 batch 下的数学形式

给定 batch 的 (N) 个样本 ((R_i,u_i))，可以构造样本版：

b^opt=∑i=1NRi∣ui∣2∑i=1N∣ui∣2\hat b_{\text{opt}} = \frac{\sum_{i=1}^{N}R_i|u_i|^2}{\sum_{i=1}^{N}|u_i|^2} b^opt​=∑i=1N​∣ui​∣2∑i=1N​Ri​∣ui​∣2​

然后用：

gi=(Ri−b^opt)uig_i=(R_i-\hat b_{\text{opt}})u_i gi​=(Ri​−b^opt​)ui​

它确实会在该 batch 上降低方差（至少降低二阶矩）。

4.4.2 batch 场景下存在的问题

(1) 它是“全局常数”，不随状态变化

LLM RLHF 的“状态”可以理解为 (x,y<t)(x,y_{<t})(x,y<t​)，不同 prompt、不同 token 位置差异极大。一个 batch 常数 baseline 混合了各种状态，离“条件最优 b∗(s)b^*(s)b∗(s)”很远。

(2) 高维代价：∣ui∣2|u_i|^2∣ui​∣2 非常贵

ui=∇θlog⁡πθ(yi)u_i=\nabla_\theta \log \pi_\theta(y_i)ui​=∇θ​logπθ​(yi​) 是十亿维梯度向量。要算范数平方就要获取并处理完整梯度结构，代价高且影响吞吐。

(3) 与同 batch 样本耦合，存在偏差与稳定性隐患

严格无偏性要求 baseline 不依赖动作；而 b^opt\hat b_{\text{opt}}b^opt​ 用了当前 batch 的 (Ri,ui)(R_i,u_i)(Ri​,ui​)，与样本存在统计耦合。大 batch 下偏差可能很小，但在实际训练（batch 不大、分布漂移快）会带来额外不稳定因素。

(4) token 级训练更需要“逐状态 baseline”（critic） RLHF 常用 token-level advantage（配合 GAE/价值网络），因为 reward 需要进行信用分配。batch 常数 baseline 无法提供逐 token 的结构化信息。

4.4.3 工业级框架的现有类似做法（“折中答案”）

工业界往往采取更稳定、性价比更高的方案：

  • Value network / Critic：学习 Vϕ(s)V_\phi(s)Vϕ​(s) 做状态相关 baseline
  • Advantage normalization：对 batch 的 advantage 做标准化

    A←A−μAσAA \leftarrow \frac{A-\mu_A}{\sigma_A} A←σA​A−μA​​

    这在数值上相当于动态缩放梯度，明显提升稳定性
  • GAE：偏差–方差折中，为 token 级 credit assignment 提供更低方差的 advantage 估计
  • KL penalty / PPO clipping：从分布漂移角度控制 ratio 尾部，抑制二阶矩

可以把 batch-optimal baseline 视为“理论上可行但不够强/不够便宜/不够稳”的方案；工业界更偏好“状态相关 critic + 归一化 + KL 约束”的组合拳。
