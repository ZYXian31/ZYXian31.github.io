旁注：

  • 如果你用的是 ($R$)（没有 baseline / advantage），并且 ($R\ge 0$) 总是成立，那么每次采样都会“强化”采样到的动作（提高其概率），这点没错。
  • 你可能会进一步问：既然这里的增减看起来与 ($A$) 的正负有关，那在没有 ($A$) 的算法里（直接用 ($R$)），而且通常 ($R\ge 0$)，是不是就意味着“无论采到好动作还是坏动作，只要采样到就一定增加它的概率”？
  • 但这并不意味着“学不到坏动作会被压下去”。原因是：softmax 的更新是耦合的，你提高某个动作的概率，会压低其他动作的概率；而“好动作被强化得更快”，“坏动作虽然也被强化但强化得更慢”，长期看坏动作会被挤出去。
  • 另外，实际算法里通常会用 baseline/advantage 的一个重要原因就是：如果 reward 永远非负，你永远只能做正向强化，方差大、学习慢、并且很容易出现错误 credit assignment。baseline 让你得到“相对好坏”（正负）信号，更新更有效。

下面将逻辑分 4 层讲透，并给一个数值例子证明“即便 ($R\ge 0$)，坏动作仍会下降”。

这意味着：每次采样到哪个动作，就把它往上推。

你这一点判断完全正确。

2) 那“坏动作”怎么会被压下去？关键在于“好动作推得更快”

虽然每次采样到的动作都会被推上去，但不同动作被采样到的频率、以及推上去的力度（由 ($R$) 决定）是不一样的。

直观地讲：

  • 好动作采样到时 ($R$) 大 ⇒ 每次推得猛
  • 坏动作采样到时 ($R$) 小（虽然仍 $\ge 0$）⇒ 每次推得弱
  • 同时好动作在推高后概率会变大 ⇒ 被采样到更频繁 ⇒ 形成正反馈
  • 坏动作概率相对减少 ⇒ 被采样到更少 ⇒ 被推高的机会更少

因此即便所有 ($R\ge 0$)，你依然会得到“相对选择”：好动作越来越占概率质量，坏动作被挤压到很小。

3) 一个具体数值例子：($R\ge 0$) 但坏动作概率仍然下降

单状态，两动作 ($A,B$)：

  • 若选 ($A$)：奖励 ($R_A = 10$)
  • 若选 ($B$)：奖励 ($R_B = 1$)（始终正的“坏动作”）

初始概率设为：

学习率 ($\alpha=0.1$)。

第 1 次采样：假设采到 B（坏动作），($R=1$)

logit 更新（用上面公式）：

  • ($\Delta z_B = 0.1\times 1 \times (1-0.5)=0.05$)
  • ($\Delta z_A = -0.1\times 1 \times 0.5=-0.05$)

这次确实“强化”了 B。

第 2 次采样：若之后采到 A（好动作），($R=10$)

此时 ($A$) 的更新力度是 B 的 10 倍：

  • ($\Delta z_A \approx 0.1\times 10 \times (1-\pi(A))$)（接近 0.5 时大约 0.5）
  • 其它动作被压下去的幅度也更大

结果通常是：只要 A 被采到几次，A 的概率会迅速接近 1，B 接近 0。

也就是说：坏动作不是因为“负奖励”才下降，而是因为“好动作得到更大的正奖励 + 更频繁被采样”，从而把概率质量挤走了。

4) 那 advantage/baseline 的价值是什么？——不是“让坏动作才会下降”，而是“让更新更高效、更低方差”

你发现了一个事实：如果 ($R\ge 0$)，更新方向只会“强化采样到的动作”。那为什么大家还要引入 advantage（允许为负）？

核心原因有两个：

(1) 把“绝对好坏”变成“相对好坏”，提高学习效率

用 ($R$) 时，你在学的是：

  • “只要有奖励就加强”

但用 advantage：

你学的是：

  • “比平均水平好就加强”
  • “比平均水平差就削弱”

这在 同一状态下区分动作时更直接，学习更快。

(2) 降低方差（这是最重要的工程原因）

即便 ($R\ge 0$)，你仍然会有很大噪声：同一个动作在不同轨迹里回报可能波动很大。baseline 把回报做中心化，显著降低估计器方差，同时不改变期望梯度。

5) 关于提到的“那不就总是提升采样到的动作吗？”——补一句更精确的话

更精确的说法是：

  • 在 没有 baseline 且 ($R\ge 0$) 的情况下，每次采样确实会提高采样到动作的 logit（局部方向为正）。
  • 但从期望意义上看，策略会更偏向“带来更高期望回报的动作”，因为它们带来的更新幅度更大、发生更频繁。

而 baseline/advantage 的作用是把这种“靠长期挤压形成的相对选择”变成“单步就有相对方向”的更新，提高样本效率并降低方差。

没有 advantage 时也能学到：只要好动作回报更高，它会被更强更频繁地强化，最终挤压坏动作的概率。
但 advantage 能更快更稳：把“绝对回报”变成“相对回报”，产生正负信号并降低方差。

严格结论

  1. 若 ($\hat A(s,a)>0$)（该动作优于基线）：

    • ($\Delta z_a>0$)，选中动作 logit 上升
    • 其余动作 logit 下降
    • 因 softmax 归一化，选中动作概率上升

  2. 若 ($\hat A(s,a)<0$)（该动作劣于基线）：

    • ($\Delta z_a<0$)，选中动作 logit 下降
    • 其余动作 logit 上升
    • 选中动作概率下降

  3. 变化幅度与 ($\hat A(s,a)$) 成正比

    • ($\hat A$) 越大，推动越强
    • ($\hat A$) 越接近 0，更新越小
    • 不存在“阈值”——只要 ($\hat A\neq 0$)，概率就会发生连续变化

在我们之前的讨论中有一个关键问题：
“奖励高到底多高才会提升概率？奖励低到底多低才会降低？”

数学上的答案非常直接：
由 advantage 的大小线性决定，没有阈值机制。

策略梯度不是分类决策规则，而是一个连续的梯度系统。

3.4 LLM 训练中方差的来源（随机变量 + 三大“方差大头”）

这一节要非常明确：方差来自哪些随机变量，以及为何在 LLM 中尤其严重。

3.4.1 明确随机变量

在 RLHF/RLAIF 中，至少包括：

  • token：$yt \sim \pi\theta(\cdot\mid y_{<t},x)$
  • 序列长度 $T$（可能是随机或由 stopping 决定）
  • 回报：$R(y)=r(y)$（reward model 给定 $y$ 通常确定，但因 $y$ 随机，$R$ 作为随机变量仍波动）
  • PPO 里还会引入 importance ratio：$r(y)=\frac{\pi\theta(y)}{\pi{\text{old}}(y)}$（见 3.5）

3.4.2 token 累积：$\sum_t u_t$ 的方差随 $T$ 增长

如 3.3 所示：即便 $R$ 不波动，$S=\sum_t u_t$ 的方差也随长度增大。

4.4.2 batch 场景下存在的问题

(1) 它是“全局常数”，不随状态变化

LLM RLHF 的“状态”可以理解为 $(x,y_{<t})$，不同 prompt、不同 token 位置差异极大。一个 batch 常数 baseline 混合了各种状态，离“条件最优 $b^*(s)$”很远。

(2) 高维代价：$|u_i|^2$ 非常贵

$ui=\nabla\theta \log \pi_\theta(y_i)$ 是十亿维梯度向量。要算范数平方就要获取并处理完整梯度结构，代价高且影响吞吐。

(3) 与同 batch 样本耦合，存在偏差与稳定性隐患

严格无偏性要求 baseline 不依赖动作；而 $\hat b_{\text{opt}}$ 用了当前 batch 的 $(R_i,u_i)$，与样本存在统计耦合。大 batch 下偏差可能很小，但在实际训练（batch 不大、分布漂移快）会带来额外不稳定因素。

(4) token 级训练更需要“逐状态 baseline”（critic）
RLHF 常用 token-level advantage（配合 GAE/价值网络），因为 reward 需要进行信用分配。batch 常数 baseline 无法提供逐 token 的结构化信息。

4.4.3 工业级框架的现有类似做法（“折中答案”）

工业界往往采取更稳定、性价比更高的方案：

  • Value network / Critic：学习 $V_\phi(s)$ 做状态相关 baseline
  • Advantage normalization：对 batch 的 advantage 做标准化这在数值上相当于动态缩放梯度，明显提升稳定性
  • GAE：偏差–方差折中，为 token 级 credit assignment 提供更低方差的 advantage 估计
  • KL penalty / PPO clipping：从分布漂移角度控制 ratio 尾部，抑制二阶矩

可以把 batch-optimal baseline 视为“理论上可行但不够强/不够便宜/不够稳”的方案；工业界更偏好“状态相关 critic + 归一化 + KL 约束”的组合拳。
