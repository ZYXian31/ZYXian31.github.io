① 49. 字母异位词分组

题目

给定字符串数组，将字母异位词分组。

示例：

题型归类

哈希分桶 / 特征签名问题

核心思想

同一组异位词一定拥有相同"签名 (key)"。

签名可以是：

  • 排序后的字符串
  • 或 26 字母计数数组

将问题从：

两两比较（O(N²)）

变为：

每个字符串生成 key，哈希分桶（O(N)）

关键数据结构

  • dict / defaultdict
  • 字符串排序
  • 或 26 长度计数数组（tuple 作为 key）

时间复杂度

  • 排序法：O(N · R log R)
  • 计数法：O(N · R)

模板代码（排序版）

可迁移题目

    1. 找到字符串中所有字母异位词
    1. 有效的字母异位词

总结：高手式解题流程

每题必须思考：

  1. 这题属于哪种模型？
  2. 能否降维？
  3. 是否可以维护不变量？
  4. 有没有排序后更简单？
  5. 是否可以用哈希避免重复计算？

当你不再记"题目"，而是记"模型"，刷题才会开始真正进阶。

附 1：通用解题推导流程

当你拿到一道题，强制问自己这 6 个问题：

① 是组合问题还是连续区间问题？

  • 选若干元素 → 组合 / 子集 / k-sum
  • 连续区间 → 滑窗 / 双指针
  • 每个位置依赖左右 → 前后缀
  • 无顺序要求 → 哈希

例如：

  • 三数之和 → 选 3 个数 → 组合问题
  • 接雨水 → 每个位置依赖左右 → 边界问题

👉 第一层分类：结构识别

② 有没有“单调性”？

  • 有序数组？
  • 可以排序吗？
  • 排序后是否更简单？

很多题一排序，结构直接暴露：

  • 三数之和 → 排序后可双指针
  • 盛水容器 → 双指针
  • 去重问题 → 排序后更简单

👉 排序是结构放大器。

④ 有没有“局部决定全局”的不变量？

例如：

  • 盛水容器 → 面积由短板决定
  • 接雨水 → 水位由较矮边界决定
  • 滑窗 → 窗口内必须满足某个条件

如果能找到“谁决定当前状态”， 基本就能双指针或贪心。

⑤ 是否存在“重复计算”？

如果有：

  • 用哈希缓存
  • 用 set 去重
  • 用记忆化
  • 用前缀和

例如：

  • 最长连续序列 → 避免重复扩展
  • 异位词分组 → 避免两两比较

⑥ 时间复杂度卡在哪里？

问自己：

  • O(n²) 会不会超？
  • O(n log n) 能不能接受？

复杂度约束会逼出结构。

例如：

  • 最长连续序列要求 O(n) → 不能排序 → 只能哈希

附 2：三数之和演示

你第一次见这题时该怎么推？

① 是组合问题（选 3 个数）

② O(n³) 太大 → 必须降维

③ 固定一个数 → 变 Two Sum

④ Two Sum 如果数组有序，可以双指针

⑤ 排序 → 去重更简单

⑥ 复杂度 O(n²) 可以接受

模型是推出来的，不是记住的。
