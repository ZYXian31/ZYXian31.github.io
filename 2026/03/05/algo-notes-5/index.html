题目二 Course Schedule（课程表）

题目描述

你这个学期必须选修 numCourses 门课程，课程编号为 0 到 numCourses - 1。

先修关系由数组 prerequisites 给出：prerequisites[i] = [ai, bi] 表示学习 ai 之前必须先完成 bi。

判断：是否可以完成所有课程？

示例：

我的初始思路（邻接矩阵检测二元环）

建立邻接矩阵 table[i][j] = 1 表示 i 的前置课程是 j，然后检测是否存在 i → k 且 k → i。

问题：这种方法只能检测长度为 2 的环。对于 0 → 1 → 2 → 0 这种三元环，矩阵中不会出现 table[0][2] 和 table[2][0] 同时为 1，所以漏判。若计算传递闭包则复杂度变成 O(n³)，在 numCourses ≤ 2000 时不可接受。

思路转换：图论建模 → 拓扑排序

课程依赖关系本质是一个有向图：bi → ai 表示 bi 是 ai 的前置课程。

问题等价于：判断有向图是否存在环。

经典结论：

所以只需尝试拓扑排序（Kahn Algorithm）：

  • 能排完所有课程 → 无环 → True
  • 不能排完 → 有环 → False

最优解法：拓扑排序（Kahn Algorithm）

步骤：

  1. 构建邻接表 adj[b].append(a)（b → a）
  2. 统计入度 indeg[a]：a 还有多少未完成的前置课程
  3. 所有入度为 0 的课程入队（可以直接学习）
  4. BFS：弹出课程，将其邻居入度 -1，入度变 0 则入队
  5. 最终若所有课程入度均为 0，说明无环

复杂度：Time O(V + E)，Space O(V + E)。

题目三 Permutations（全排列）

题目描述

给定一个不含重复数字的数组 nums，返回其所有可能的全排列。你可以按任意顺序返回答案。

示例：

限制：1 <= nums.length <= 6，nums 中所有整数互不相同。

我的第一反应：这是典型回溯（Backtracking）

"全排列"本质是在构造一个长度为 n 的序列：每个位置都要放一个元素，每个元素只能用一次。

框架：

解法一：通用模板（回溯 + used 数组）【最推荐】

  • path：当前正在构造的排列
  • used[i]：nums[i] 是否已经放进 path
  • 当 len(path) == n，一个排列完成，加入答案

复杂度：时间 O(n * n!)，空间 O(n)（递归栈）+ O(n * n!)（输出）。

我踩过的坑：忘了写 copy()

错误写法：

因为 path 是同一个列表对象，递归过程中不断被 append/pop 修改。把引用塞进 res，最终所有元素指向同一个对象，结果通常全部是空列表 [] 或最后一次的状态。

diff

关键：必须把当前路径的快照存起来。

解法二：原地交换（swap）法

用 nums 本身充当暂存区，first 指针表示"当前位置已确定到哪里"：

把 nums[first] 与 nums[i] 交换，就表示"选择 nums[i] 放到当前位置"；递归后换回来恢复现场。

为什么一定要换回来？ 如果不换回，数组持续被污染，后续分支就不是在同一个起点状态上探索了。回溯的本质就是：做选择 → 递归 → 撤销选择。

两种模板对比

方法 暂存区 优点 适合
used + path path 独立列表 思路直观，最易加剪枝/条件 大多数排列/回溯题
swap 交换 nums 原地充当暂存区 代码短，少一个 used 数组 纯排列、省额外结构

建议先熟练 used + path，遇到"排列 + 去重 + 条件"更好改。

总结

  1. 全排列是标准回溯：每层选一个未用元素，构造长度 n 的路径
  2. 两个模板：used + path（最通用）和 swap（原地回溯）
  3. 最常踩的坑：忘记 copy()，无论保存 path 还是 nums 都要快照
  4. 回溯三步口诀：做选择 → dfs → 撤销选择

题目四 Subsets（子集 / 幂集）

这题是回溯里最"模板化"的一类：子集 / 幂集。抽象成一句话就是：

子集 = 对每个元素只有两种选择：选 或 不选。

模板 1：最通用回溯（推荐，最好背）

核心：从下标 start 开始，决定后面选哪些数；每到一个节点就把当前 path 加入答案（因为 path 本身就是一个子集）。

复杂度：结果数量 2^n，时间 O(n * 2^n)（复制 path 的成本），空间 O(n)（递归深度）。

dfs(start) 的真正语义

  • start 保证每个元素最多选一次，且不会出现 [1,2] 和 [2,1] 这种重复组合
  • 每次进入 dfs 都 res.append(path)：因为"到达的每个节点"都代表一种合法子集（包括空集）

附录：回溯算法完全指南（子集 / 组合 / 排列 一篇掌握）

回溯的 统一模型：90% 的回溯题都来自三个模板——子集、组合、排列。

一、回溯算法的本质

回溯 = 暴力搜索 + 剪枝。

关键问题只有两个：当前可选集合是什么 / 递归状态如何更新。

二、三大核心模板

子集（Subsets）

触发词：subset / power set / 所有子集。特点：每一个节点都是答案（子集数量 2^n）。

dfs(start) 语义：当前已选 path，接下来从 nums[start:] 继续选。dfs(i + 1) 保证每个元素最多选一次，避免 [1,2] 和 [2,1] 重复。

组合（Combination）

触发词：组合 / 选 k 个 / 和为 target。特点：只有满足条件的节点才加入答案。

排列（Permutation）

触发词：permutation / 全排列 / 顺序不同算不同。特点：用 used 标记已使用元素，每层从 0 开始选。

三、三大模板对比

类型 参数 选择范围 何时加入答案
子集 start start → n 每个节点
组合 start start → n 满足条件时
排列 used 0 → n len(path) == n

这类题常见的回溯写法有两套“范式”，选一种写到底就不会乱：

范式 A：for 循环枚举下一步选谁（推荐）

特点：

Bug 1：递归会越界（start 可能超过 n）
